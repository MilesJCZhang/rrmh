## 一、生产环境实际技术栈

### 前端技术栈（服务器实际部署版本）
- **构建工具**: Vite 7.2.4
- **框架**: React 19.2.0 + TypeScript ~5.9.3

### 活动管理现状（实际代码分析）

1. **统计卡片完整**: 活动总数、报名中、已结束、今日活动、参与人数、活动收入

2. **活动类型单一**: 只有"线下沙龙(1)"和"线上活动(2)"，没有男女推荐官沙龙类型
3. **缺少推荐官管理**: 没有专门的推荐官业绩页面

6. **海报管理简单**: 只能查看链接，无法预览
7. **拒绝原因未保存**: 代码中未见 reject_reason 相关处理

#### 1.1 添加活动审核功能
**目标**: 为活动管理添加审核通过/拒绝功能

**技术方案**:
```typescript
// 1. 在 lib/activityApi.ts 中添加审核API
export async function approveActivity(
  id: number,
  action: 'approve' | 'reject',
  rejectReason?: string
): Promise<void> {
  await request(`/v1/salon/${id}/approve`, {
    method: 'PUT',
    body: JSON.stringify({ action, reject_reason: rejectReason }),
  });
}

// 2. 在 pages/activities/index.tsx 中添加审核按钮和弹窗
const [approveVisible, setApproveVisible] = useState(false);
const [rejectVisible, setRejectVisible] = useState(false);
const [rejectReason, setRejectReason] = useState('');

const handleApprove = (record: Activity) => {
  Modal.confirm({
    title: '审核通过',
    content: `确定通过活动"${record.title}"吗？`,
    onOk: async () => {
      await approveActivity(record.id, 'approve');
      message.success('审核已通过');
      loadList();
    },
  });
};

const handleReject = (record: Activity) => {
  setSelectedActivity(record);
  setRejectVisible(true);
};

// 3. 在表格的操作列中添加按钮
{
  title: '操作',
  render: (_, record) => (
    <Space>
      {record.status === 0 && (  // 待审核状态
        <>
          <Button type="link" onClick={() => handleApprove(record)}>
            通过
          </Button>
          <Button type="link" danger onClick={() => handleReject(record)}>
            拒绝
          </Button>
        </>
      )}
      <Button type="link" onClick={() => handleViewDetail(record)}>
        详情
      </Button>
    </Space>
  ),
}
```
