## 推荐官审核功能不显示问题修复（2026-05-28 第二次部署）

### 问题
管理后台「专业推荐官」页面看不到审核按钮，API 返回 401。

### 根因分析
1. **路由冲突**：`admin.ts` 和 `admin-users.ts` 都注册了 `/officers` 路由在 `/api/admin` 下
   - `admin-users.ts` 的 `/officers` 有 `authMiddleware`，返回 401
   - Express 按 `app.use` 注册顺序匹配，但 admin-users 的认证中间件拦截了请求
2. **前端 API 路径**：`userApi.ts` 调用 `/admin-api/officers` → nginx rewrite → `/api/admin/officers`
3. **编译失败**：本地 `admin.ts` 缺少服务器上特有的 import（activityTemplateController 等），scp 覆盖后编译报错

### 修复内容
1. **删除 `admin-users.ts` 中的 `/officers` 路由段**（第 170-341 行），避免与 `admin.ts` 冲突
2. **重写 `admin.ts`**：基于服务器实际存在的 controller 文件，保留 officers 别名路由
3. **重新构建前端**：`admin-src/` 中 `npm run build` 后部署到 `/var/www/admin/`
4. **正确的部署路径**：`/var/www/admin/`（nginx alias），非 `/home/ubuntu/.../public/admin/`

### 验证结果
- ✅ `GET /api/admin/officers` → 200, total=25
- ✅ `GET /api/admin/recommend-officers` → 200, total=25
- ✅ `PUT /api/admin/officers/:id/review` → 200, "已通过审核"
- ✅ `GET https://rrmhdate.cn/admin-api/officers` → 200（外网 nginx 代理）
- ✅ `PUT https://rrmhdate.cn/admin-api/officers/30/review` → 200（外网审核）
- ✅ 前端 `index-DtsU4PiX.js` 包含 `officers/${id}/review` 审核 API 调用

### 关键教训
- 服务器上 controller 文件和本地不一致，需要先确认服务器环境再修改
- `admin-users.ts` 和 `admin.ts` 有路由冲突，不要在两个文件定义同一路径
- 管理后台部署路径是 `/var/www/admin/` 而非 `/home/ubuntu/renrenmeihao-api/public/admin/`

## switch-role 接口 504 修复（2026-05-28 第六次修复）

### 问题
小程序端身份切换调用 `/v1/user/switch-role` 超时，nginx 返回 504。

### 根因
`src/routes/user.ts` 中 switch-role 路由的 `validate` 调用缺少 `[]`：
- 正确: `validate([])` — 调用工厂函数，返回中间件
- 错误: `validate` — 传入工厂函数本身，Express 无法正确处理

Express 收到非标准签名的中间件时挂起，导致请求永远不会被响应，nginx 60s 后返回 504。

### 修复
`src/routes/user.ts` 第326行：`validate` → `validate([])`

### 验证
- ✅ 本地测试 `http://127.0.0.1:3001/v1/user/switch-role` 返回 200
- ✅ 身份切换正常（city_franchisee ↔ single）

`pages/mine/mine.js` `onRoleSwitch()` 中用 `new Promise` 包装 `request()` 调用，并传入 `success`/`fail` 回调。但 `request.js` 返回的是原生 Promise（用 `resolve`/`reject`），**不会调用 `options.success`/`options.fail`**，导致外层 Promise 永远 pending。

### 修复

## 推荐官管理洞察数据不显示修复（2026-05-28 第八次修复）

### 问题
城市合伙人 CSHH83UR (userId=3) 推荐了公益推荐官 GYRG4VW9 (userId=97)，但在推荐官管理后台的洞察数据中没有体现。实际上所有推荐官的洞察数据（totalReferrals, firstReferralRecord, referralCode 等）都是空的。

### 根因
`recommendOfficerController.adminList` 接口只返回了 `recommendOfficer` 基本字段 + `user` 信息，**完全没有计算洞察数据**。前端表格需要 `referralCode`, `totalReferrals`, `totalEarnings`, `availableBalance`, `passiveEarnings`, `firstReferralRecord` 这些字段。

更深层的原因：`recommendations` 表完全为空（0 条记录）。推荐关系是通过 `users.recommendedBy` 字段建立的（微信登录/注册时），但 `createRecommendation()` 没有被调用写入 `recommendations` 表。

### 修复方案
修改 `adminList` controller，基于 `users.recommendedBy` 为主数据源计算洞察数据：
1. **推荐码**：从 `referral_codes` 表取（已有数据）
2. **推荐人数**：`SELECT COUNT(*) FROM users WHERE recommendedBy = userId`
3. **首推记录**：按被推荐用户的 `role` 字段细分统计
   - `public_matchmaker` → member（公益推荐官）
   - `partner_matchmaker` / `professional_recommender` → partner（联创推荐官）
   - `community_station` → station（社区服务站）
   - `city_franchisee` → franchisee（城市合伙人）
   - `user` + 已建档 → single（单身建档）
4. **收益数据**：从 `wallets.totalIncome` 获取（同时保留 `recommendations` 表作为辅助数据源）
