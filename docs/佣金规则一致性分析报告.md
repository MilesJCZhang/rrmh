## 四、发现的问题与建议

### ✅ 一致性确认

1. **费用标准**: 所有费用标准（199、399、3999、10000）前后端一致 ✅
2. **收益比例**: 推荐建档99元、沙龙补贴99元等核心比例一致 ✅
3. **首推规则**: "第2个起才有佣金"逻辑前后端一致 ✅
4. **v5.1新增规则**: 社区服务站沉淀资金分配逻辑已正确实现 ✅

### ⚠️ 需要确认的点

1. **沙龙收益计算细节**:
   - 前端显示主办方到手200元/人
   - 后端逻辑是发放两份99元补贴（承办人+联创推荐人）
   - **建议**: 明确200元的组成（票价399 - 平台100 - 联创补贴99 = 200元）✅ 逻辑正确

2. **提现手续费计算位置**:
   - 前端 `commissionRules.js` 中有完整的提现手续费计算函数
   - 后端 `commission_engine.js` 中没有明确体现
   - **建议**: 确认提现手续费是在前端计算还是后端计算，避免重复或遗漏

3. **城市合伙人沉淀资金**:
   - 前端: `withdrawalFeeRate: 0` (提现直接拿净额)
   - 后端: 70%直接记入 `platform_fund` 表，状态为 `accumulated`
   - **建议**: 确认城市合伙人提现时是否需要再扣除费用（目前逻辑是不需要）✅ 一致

### 🔧 代码层面的小问题

1. **后端 `REFERRER_ELIGIBLE_ROLES`** (commission_engine.js:68):
   ```javascript
   const REFERRER_ELIGIBLE_ROLES = ['public_matchmaker', 'partner_matchmaker', 'community_station'];
   ```
   - 这个数组决定了哪些角色可以获得"推荐建档佣金"
   - ✅ 正确包含了三个角色
   - ⚠️ 但**不包含** `professional_recommender` 和 `city_franchisee`，这意味着专业推荐官和城市合伙人推荐普通用户建档是**没有佣金**的
   - **确认**: 根据v5.1规则，专业推荐官和城市合伙人推荐建档应该也得99元，建议确认是否应该把他们也加入这个数组

2. **后端 `CITY_REFERRER_ELIGIBLE_ROLES`** (commission_engine.js:71):
   ```javascript
   const CITY_REFERRER_ELIGIBLE_ROLES = ['professional_recommender'];
   ```
   - ✅ 正确：只有专业推荐官可以推荐城市合伙人并获得佣金
   - 符合v5.1"独家推荐权"的规则

## 五、总结

### ✅ 主要结论

**小程序当前的佣金计算方式与v5.1分润规则基本一致**，没有发现严重的错乱。

核心规则（费用标准、收益比例、首推限制、v5.1新增的社区服务站分配）都已经正确实现。

### 🔧 建议修复的小问题

1. **考虑将 `professional_recommender` 和 `city_franchisee` 加入 `REFERRER_ELIGIBLE_ROLES`**，使专业推荐官和城市合伙人也能获得推荐建档的99元佣金（如果他们推荐普通用户建档）。

2. **明确提现手续费的计算位置**，避免前后端重复计算或遗漏。

3. **添加单元测试**，覆盖所有角色的所有收益场景，防止未来代码修改时破坏一致性。
