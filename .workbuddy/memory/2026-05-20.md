## 第二轮修复（同日上午）

### 新问题：保存微信号 403 + 身份写入失败

**现象**：
- PUT `/v1/user/profile/update` 返回 403
- 服务端日志：`Invalid prisma.user.update() invocation`，`role` 字段写入失败

**根因 1：403 误判**

`/v1/user/profile/update` 接口中，403 检查逻辑在占位用户合并逻辑**之后**，但检查 `existingUserWithWechat` 时没有排除 `openid=null` 的占位用户。如果占位用户删除失败（`.catch(() => {})` 吞掉错误），就会误报 403。

修复（`src/routes/user.ts`）：
```typescript
// 修改前
const existingUserWithWechat = await prisma.user.findFirst({
  where: {
    wechatAccount: wechatAccount,
    NOT: { id: userId },
  },
});

// 修改后
const existingUserWithWechat = await prisma.user.findFirst({
  where: {
    wechatAccount: wechatAccount,
    NOT: { id: userId },
    openid: { not: null },  // 排除占位用户
  },
});
```

**根因 2：role 字段长度超限**

Prisma schema 中 `User.role` 定义：`@db.VarChar(20)`，但 `'professional_recommender'` 有 24 个字符，写入时报 `P2000` 错误。

修复（`prisma/schema.prisma`）：
```
// 修改前
role  String  @default("single") @db.VarChar(20)

// 修改后
role  String  @default("single") @db.VarChar(50)
```
执行 `npx prisma db push` 同步到数据库。

### 修改的文件（服务端）
- `/home/ubuntu/renrenmeihao-api/src/routes/user.ts` — 403 检查排除占位用户
- `/home/ubuntu/renrenmeihao-api/prisma/schema.prisma` — role 字段改为 VarChar(50)
- 重新 `npm run build` + `pm2 restart renrenmeihao-api`

**排查过程**：
1. 服务端 `/insight/:code` 直接 curl 测试返回 200（说明路由本身没问题）

4. nginx rewrite `/admin-api/(.*)` → `/api/admin/$1` 后，Express `app.use('/api/admin/referral-codes', adminReferralRoutes)` 将前缀去掉，router 实际收到的是 `/CSHH83UR/insights`

**修复**（`src/routes/admin-referral.ts`）：
- 将洞察处理逻辑提取为 `handleInsight` 函数
- 注册两条路由：`router.get('/:code/insights', handleInsight)` 和 `router.get('/:code/insight', handleInsight)`
- 同时兼容前端实际请求的 `/insights`（复数）和源码中定义的 `/insight`（单数）

## 踩坑经验积累

### Prisma 相关

1. **字段名映射**：DB 用 `created_at`（snake_case），Prisma schema 用 `createdAt @map("created_at")`，JS/TS 代码中用 `createdAt`（camelCase）。
2. **关系字段名**：多个同名关系时，Prisma 自动生成长名如 `User_ReferralRelationship_refereeIdToUser`，必须通过 `@relation("...")` 自定义短名，或查看生成的 `.d.ts` 文件确认。
3. **`prisma generate` 二进制目标**：Linux 服务器需要 `binaryTargets = ["native", "debian-openssl-3.0.x"]`，否则生成的 Client 不能在服务器运行。

### Express 路由相关

4. **`Router.use()` 报错 `undefined`**：通常是路由文件导出语句缺失或导出格式错误（CommonJS vs ESM）。
5. **TypeScript `export default` 编译**：编译后变成 `exports.default = xxx; module.exports = xxx;`，`require()` 后必须用 `.default` 访问。

### 调试技巧

6. **PM2 启动失败排查**：
   - `pm2 logs <name> --lines 50 --nostream` 查看错误日志
   - `node -c dist/routes/xxx.js` 检查语法
   - `ssh ubuntu@... 'curl -s -o /dev/null -w "%{http_code}\n" http://localhost:3001/health'` 测试服务是否监听端口
7. **数据库列名不一致**：用 `SHOW COLUMNS FROM users;` 确认实际列名，不要用 Prisma schema 猜测。

### 微信小程序相关

8. **实际运行的页面**：Console 日志 `[Perf][pages/xxx/xxx] Page.onShow` 显示实际运行的页面路径，不要用假设。
9. **`request()` 返回值**：如果用了响应拦截器自动提取 `body.data`，后续代码直接用 `res` 而不是 `res.data`。
