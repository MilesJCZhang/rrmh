## 需求概述

将管理后台的更新（沙龙活动审核的"拒绝原因"功能）部署到生产服务器 https://rrmhdate.cn/admin/

## 已完成工作

1. ✅ 管理后台代码已修改完成（Activities/index.tsx 和 activity.service.ts）
2. ✅ 本地构建已完成（admin-panel/build/ 目录已生成）
3. ✅ 构建产物验证通过（包含 reject_reason 相关代码）

## 部署目标

- 生产服务器：rrmhdate.cn
- 管理后台路径：https://rrmhdate.cn/admin/
- 需要重启生产服务器以加载新的管理后台静态文件

## 部署方式

需要通过SSH将构建文件上传到生产服务器，并重启服务器

## 技术栈

- 前端：React + TypeScript + Ant Design
- 构建工具：react-scripts
- 后端：Node.js + Express
- 部署方式：SSH/SFTP 上传静态文件

## 部署架构

```
本地构建 (admin-panel/build/)
    ↓ (SSH上传)
生产服务器 (rrmhdate.cn:/path/to/admin-panel/build/)
    ↓ (重启服务器)
服务生效 (https://rrmhdate.cn/admin/)
```

## 关键技术点

1. **静态文件服务**：server.js 已配置 `app.use('/admin', express.static(adminBuildPath))`
2. **SPA fallback**：已配置所有非API请求返回 index.html
3. **构建产物**：build/ 目录包含 index.html, static/, asset-manifest.json

## 实施步骤

1. 准备部署包：确认 admin-panel/build/ 目录完整
2. 连接生产服务器：需要SSH凭证
3. 上传构建文件：使用 scp 或 rsync 上传到服务器
4. 重启服务器：重启 Node.js 服务以加载新的静态文件
5. 验证部署：访问 https://rrmhdate.cn/admin/ 验证功能

## 风险控制

- 备份当前生产版本
- 在维护窗口期部署
- 准备回滚方案

## Agent Extensions

由于需要部署到生产服务器，建议使用以下扩展（如果可用）：

- **ssh**: 用于SSH连接和文件上传
- **filesystem**: 用于准备部署包和验证文件

注意：当前环境中可能没有配置SSH扩展，需要用户提供服务器登录信息或通过其他方式部署。
