# 沙龙配置管理功能 - 快速部署指南

## 一、后端部署（服务器）

### 1. 上传修改后的文件到服务器

将以下文件上传到服务器（`ubuntu@175.24.227.251:/home/ubuntu/renrenmei/miniprogram/`）：

```bash
# 在本地项目目录执行
scp routes_admin_salon_config.js ubuntu@175.24.227.251:/home/ubuntu/renrenmei/miniprogram/
scp server.js ubuntu@175.24.227.251:/home/ubuntu/renrenmei/miniprogram/
scp create_tables.js ubuntu@175.24.227.251:/home/ubuntu/renrenmei/miniprogram/
```

### 2. 在服务器上创建数据库表

登录到服务器，执行以下命令：

```bash
ssh ubuntu@175.24.227.251
cd /home/ubuntu/renrenmei/miniprogram
node create_tables.js
```

这将创建 `salon_configs` 表并插入默认配置。

### 3. 重启后端服务

```bash
# 如果使用 PM2
pm2 restart all

# 或者查看进程并重启
pm2 list
pm2 restart <进程名>

# 如果没有使用进程管理，需要手动重启
# 先杀掉进程
pkill -f "node server.js"
# 然后重新启动
nohup node server.js > server.log 2>&1 &
```

## 二、管理后台部署

### 1. 构建管理后台

在本地项目目录执行：

```bash
cd /Volumes/User/MacBookAir/人人媒好/相亲小程序/miniprogram/admin-panel
npm run build
```

### 2. 上传构建文件到服务器

将 `build/` 目录上传到服务器的管理后台目录：

```bash
# 使用构建脚本（推荐）
cd /Volumes/User/MacBookAir/人人媒好/相亲小程序/miniprogram/admin-panel
./build.sh

# 或者手动上传
scp -r build/* ubuntu@175.24.227.251:/var/www/html/admin-panel/build/
```

### 3. 验证管理后台

访问 `http://rrmhdate.cn/admin/login`，使用管理员账号登录，检查左侧菜单是否有 **沙龙配置** 菜单项。

## 三、小程序端部署

### 1. 上传修改后的文件

将 `utils/salon-config.js` 上传到小程序项目（如果使用版本管理，直接提交代码即可）。

### 2. 在小程序中初始化配置

在 `app.js` 中添加初始化代码：

```javascript
// app.js
const { initSalonConfig } = require('./utils/salon-config');

App({
  onLaunch() {
    // 初始化沙龙配置
    initSalonConfig();
  },
});
```

### 3. 测试

在微信开发者工具中测试：

1. 打开小程序
2. 进入沙龙列表页面
3. 检查是否正常显示
4. 在管理后台修改配置，然后重启小程序，检查配置是否更新

## 五、常见问题

### 1. 后端启动报错：Cannot find module './routes_admin_salon_config'

**解决方法**：检查 `routes_admin_salon_config.js` 文件是否上传到服务器，并且路径是否正确。

### 2. 管理后台页面空白

**解决方法**：
1. 检查浏览器控制台是否有错误
2. 检查 `admin-panel/build/` 目录是否正确上传
3. 检查 Nginx 配置是否正确

### 3. 小程序端无法获取配置

**解决方法**：
1. 检查后端API是否正常（使用 curl 测试）
2. 检查小程序端 `API.BASE_URL` 是否正确
3. 检查网络连接是否正常
