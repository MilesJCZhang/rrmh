## 修复微信小程序后端API 404错误

### 问题描述
用户报告小程序启动时遇到多个404错误，无法访问以下API端点：
1. `GET /config/public-map` - 404
2. `POST /auth/wechat-login` - 404
3. `GET /api/admin/referral-codes/list` - 404
4. `POST /api/admin/referral-codes/generate` - 404
5. `POST /api/admin/referral-codes/manual-create` - 404

### 根本原因
1. **路由未注册**：`routes_config.js` 在 `server.js` 中没有注册
2. **路径不匹配**：
   - `routes_auth.js` 注册为 `/api/auth`，但前端请求 `/auth/wechat-login`
   - `routes_referral-codes.js` 注册为 `/api/referral-codes`，但前端请求 `/api/admin/referral-codes/*`
3. **路由缺失**：`/list` 和 `/manual-create` 路由在 `routes_referral-codes.js` 中未定义

### 解决方案

#### 1. 修改 `server.js` 路由注册（第22-48行）
```javascript
// 1. 配置路由（必须放在认证路由之前）
app.use('/config', require('./routes_config'));

// 2. 认证路由
app.use('/auth', require('./routes_auth'));

// 3. 用户路由
app.use('/api/user', require('./routes_user') || createUserRoutes());

// 4. 推荐码管理路由（管理端）
app.use('/api/admin/referral-codes', require('./routes_referral-codes'));
```

#### 2. 在 `routes_referral-codes.js` 中添加缺失路由

**添加 `/list` 路由**（第12-84行）：
- 路径：`GET /api/admin/referral-codes/list`
- 功能：获取推荐码列表，支持分页和筛选（user_id, code_type, status）
- 返回：推荐码数组 + 分页信息

**添加 `/manual-create` 路由**（第86-147行）：
- 路径：`POST /api/admin/referral-codes/manual-create`
- 功能：手动创建推荐码，支持指定码值或自动生成
- 参数：user_id, code_type, code (可选)
- 返回：创建的推荐码信息

### 技术要点
1. **路由注册顺序很重要**：config路由必须放在auth路由之前，因为小程序启动时会先加载配置
2. **路径匹配要精确**：前端请求路径必须与后端注册路径完全匹配
3. **数据库操作使用 better-sqlite3**：注意语法是同步的，使用 `.get()` 和 `.all()` 方法
4. **错误处理的业务码**：使用 `success: false` + `message` 而不是HTTP状态码
