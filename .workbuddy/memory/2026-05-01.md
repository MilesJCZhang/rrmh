## 支付 JSAPI 「缺少参数 total_fee」深层修复（09:39）

### 真实根因（此前误判）
上述修复均非真正原因。真实问题链如下：
1. **服务端 `routes/auth.js` 登录接口未接入真实微信**：`wechat-login` 接口里 TODO 注释未实现，`openid` 直接 Mock 生成 `dev_xxx` 假值，未调用微信 `code2session`
2. **所有用户的 openid 均为 `dev_` 前缀假数据**（数据库已证实）
3. **微信统一下单 API 收到非法 openid → `result_code=FAIL`, `err_code=PARAM_ERROR`**，但服务端只判断了 `return_code`，未判断 `result_code`，把失败当成功返回
4. **前端拿到 `prepay_id=undefined`，调 `wx.requestPayment({package: 'prepay_id=undefined'})` → 微信弹出「缺少参数 total_fee」**（误导性报错）

### 修复内容
1. **服务端 `routes/auth.js`**：`wechat-login` 接口改为真实调用 `https://api.weixin.qq.com/sns/jscode2session`，获取真实 openid 和 session_key
2. **服务端 `routes/payment.js`**：在 `return_code` 校验后补加 `result_code` 校验，失败时打印微信错误并返回 400（不再把失败当成功）
3. 服务已 pm2 restart，在线运行

### 说明
- 修复后，登录将产生真实微信 openid（`o` 开头），历史 `dev_` 用户需重新登录（会建新账号）
- `WECHAT_APP_SECRET=e82e162f...` 已于 09:35 更新，code2session 调用应能正常工作

## 修复沙龙报名支付状态问题

### 问题
沙龙报名支付后，预约状态没有正确更新。代码中使用 `'pending'` 和 `'confirmed'` 状态，但数据库枚举值只支持 `'registered','attended','cancelled'`。

### 解决方案
1. **修改数据库表结构**：
   ```sql
   ALTER TABLE salon_registrations 
   MODIFY COLUMN status ENUM('pending_payment','registered','attended','cancelled') 
   NOT NULL DEFAULT 'pending_payment'
   ```

2. **修改代码状态值**：
   - `routes/salon.js`：将 `'pending'` → `'pending_payment'`，`'confirmed'` → `'registered'`
   - `routes/payment.js`：将 `SET status='confirmed'` → `SET status='registered'`

3. **重启 PM2 服务**：`pm2 restart renrenmei-api`

### 业务流程
```
用户报名 → 创建报名记录(status='pending_payment')
    ↓
调用支付 → 创建订单(status='pending')
    ↓
支付成功 → 微信回调通知
    ↓
更新订单(status='paid') + 更新报名(status='registered') + 沙龙人数+1
```

### 关键文件
- 数据库：`salon_registrations`、`salons`、`orders` 表
- 后端：`routes/salon.js`、`routes/payment.js`
- 配置：PM2 环境变量 JWT_SECRET
