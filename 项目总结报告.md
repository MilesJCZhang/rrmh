## 一、项目概述

### 1.1 项目背景
人人媒好是一个基于微信小程序的相亲交友平台，旨在为用户提供匹配、交友、婚恋服务。项目采用前后端分离架构，后端提供RESTful API，前端包括微信小程序和React管理后台。

### 1.2 技术栈
- **后端框架**：Node.js + Express.js
- **数据库**：SQLite 3 + better-sqlite3
- **前端小程序**：微信小程序原生开发
- **管理后台**：React 18 + Create React App
- **认证方式**：JWT（待完整实现）
- **API风格**：RESTful

## 三、数据库设计

### 3.1 数据库初始化
**文件**：`database.js`

**初始化顺序**：

2. stations 表（新增）
3. partners 表（新增）

### 3.2 表结构设计

#### 3.2.1 stations 表（相亲服务站）
```sql
CREATE TABLE IF NOT EXISTS stations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                    -- 服务站名称
  address TEXT,                          -- 详细地址
  city TEXT,                            -- 城市
  phone TEXT,                           -- 联系电话
  description TEXT,                      -- 服务描述
  images TEXT,                          -- 图片列表（JSON数组）
  business_hours TEXT,                  -- 营业时间
  rating DECIMAL(2,1) DEFAULT 5.0,     -- 评分
  status TEXT DEFAULT 'active',          -- 状态
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

**索引**：
```sql
CREATE INDEX idx_stations_city ON stations(city);
CREATE INDEX idx_stations_status ON stations(status);
```

#### 3.2.2 partners 表（合伙人）
```sql
CREATE TABLE IF NOT EXISTS partners (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL UNIQUE,       -- 关联用户ID（唯一）
  type TEXT NOT NULL,                    -- 合伙人类型
  id_card TEXT,                          -- 身份证号
  id_card_front TEXT,                    -- 身份证正面照
  id_card_back TEXT,                     -- 身份证反面照
  status TEXT DEFAULT 'pending',          -- 审核状态
  total_earnings DECIMAL(10,2) DEFAULT 0, -- 总收益
  available_commission DECIMAL(10,2) DEFAULT 0, -- 可用佣金
  frozen_commission DECIMAL(10,2) DEFAULT 0, -- 冻结佣金
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
)
```

**索引**：
```sql
CREATE INDEX idx_partners_user_id ON partners(user_id);
CREATE INDEX idx_partners_status ON partners(status);
```

#### 3.2.3 withdrawals 表（提现申请）
```sql
CREATE TABLE IF NOT EXISTS withdrawals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,              -- 用户ID
  amount DECIMAL(10,2) NOT NULL,        -- 提现金额
  bank_account TEXT NOT NULL,            -- 银行卡号
  bank_name TEXT NOT NULL,               -- 开户银行
  account_holder TEXT NOT NULL,          -- 开户人姓名
  status TEXT DEFAULT 'pending',         -- 审核状态
  admin_remark TEXT,                     -- 管理员备注
  processed_at DATETIME,                 -- 处理时间
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
)
```

**索引**：
```sql
CREATE INDEX idx_withdrawals_user_id ON withdrawals(user_id);
CREATE INDEX idx_withdrawals_status ON withdrawals(status);
CREATE INDEX idx_withdrawals_created_at ON withdrawals(created_at);
```

// 获取服务站列表（支持分页和筛选）
router.get('/', (req, res) => {
  const { city, status, page = 1, limit = 20 } = req.query;
  const offset = (page - 1) * limit;
  
  let query = 'SELECT * FROM stations WHERE 1=1';
  const params = [];
  
  if (city) {
    query += ' AND city = ?';
    params.push(city);
  }
  
  if (status) {
    query += ' AND status = ?';
    params.push(status);
  }
  
  query += ' ORDER BY created_at DESC LIMIT ? OFFSET ?';
  params.push(parseInt(limit), parseInt(offset));
  
  const stations = db.prepare(query).all(...params);
  
  res.json({
    success: true,
    data: { stations, pagination: { page, limit, total, total_pages } }
  });
});

### 5.2 合伙人路由（routes_partners.js）

**特点**：
- 申请接口 `/apply` 独立设计
- 处理UNIQUE约束冲突
- 收益查询使用LEFT JOIN（简化版本）

**关键代码片段**：
```javascript
// 申请成为合伙人
router.post('/apply', (req, res) => {
  const { user_id, type, id_card, id_card_front, id_card_back } = req.body;
  
  // 参数验证
  if (!user_id || !type) {
    return res.status(400).json({
      success: false,
      message: '用户ID和类型为必填项'
    });
  }
  
  try {
    const stmt = db.prepare(`
      INSERT INTO partners (user_id, type, id_card, id_card_front, id_card_back)
      VALUES (?, ?, ?, ?, ?)
    `);
    
    const info = stmt.run(user_id, type, id_card || null, id_card_front || null, id_card_back || null);
    
    const partner = db.prepare('SELECT * FROM partners WHERE id = ?').get(info.lastInsertRowid);
    
    res.status(201).json({
      success: true,
      data: partner,
      message: '申请提交成功，等待审核'
    });
  } catch (error) {
    // 处理UNIQUE约束冲突
    if (error.message.includes('UNIQUE constraint failed')) {
      return res.status(400).json({
        success: false,
        message: '您已经申请过成为合伙人'
      });
    }
    throw error;
  }
});
```

### 5.3 提现路由（routes_withdrawals.js）

**特点**：
- 使用 `requireAuth` 中间件（暂时简化）
- 提现前检查可用佣金
- 提现后冻结佣金
- 取消提现时解冻佣金

**关键代码片段**：
```javascript
// 申请提现
router.post('/', requireAuth, (req, res) => {
  const db = req.app.get('db');
  const { user_id, amount, bank_account, bank_name, account_holder } = req.body;
  
  try {
    // 检查用户是否存在
    const user = db.prepare('SELECT * FROM users WHERE id = ?').get(user_id);
    if (!user) {
      return res.status(404).json({ success: false, message: '用户不存在' });
    }
    
    // 检查可用佣金是否足够
    if (user.available_commission < amount) {
      return res.status(400).json({
        success: false,
        message: '可用佣金不足'
      });
    }
    
    // 创建提现申请
    const stmt = db.prepare(`
      INSERT INTO withdrawals (user_id, amount, bank_account, bank_name, account_holder, status)
      VALUES (?, ?, ?, ?, ?, 'pending')
    `);
    
    const info = stmt.run(user_id, amount, bank_account, bank_name, account_holder);
    
    // 冻结佣金
    db.prepare('UPDATE users SET available_commission = available_commission - ? WHERE id = ?')
      .run(amount, user_id);
    
    const withdrawal = db.prepare('SELECT * FROM withdrawals WHERE id = ?').get(info.lastInsertRowid);
    
    res.status(201).json({
      success: true,
      data: withdrawal,
      message: '提现申请提交成功'
    });
  } catch (error) {
    // 错误处理
  }
});
```

**示例代码**：
```javascript
const jwt = require('jsonwebtoken');

const requireAuth = (req, res, next) => {
  const token = req.headers.authorization?.replace('Bearer ', '');
  
  if (!token) {
    return res.status(401).json({ success: false, message: '未提供认证令牌' });
  }
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ success: false, message: '认证令牌无效' });
  }
};

module.exports = { requireAuth };
```
