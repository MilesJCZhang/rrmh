# 推荐官工作台功能升级 - 实施总结

## 一、需求概述

在现有 `matchmaker-workbench` 页面基础上扩展功能，新增专属数据看板，自动统计通过本人专属码/链接引流的全部人员，分类展示累计人数与人员明细。

## 二、已完成工作

### 1. 扩展 `api.js` - 新增工作台数据看板相关 API 端点
**文件**：`/services/api.js`

新增以下端点：
- `REFERRAL.WORKBENCH_STATS`: `/v1/referral/workbench-stats`（GET 四大分类统计数据）
- `REFERRAL.WORKBENCH_DETAIL`: `/v1/referral/workbench-detail`（GET 分类明细列表，支持分页、搜索）
- `REFERRAL.VISITOR_LOG`: `/v1/referral/visitor-log`（POST 记录访客到访）
- `REFERRAL.VISITOR_UPDATE`: `/v1/referral/visitor-update`（PUT 更新访客注册状态）

- `logVisitor(data)`: 记录访客到访

### 3. 修改 `matchmaker-workbench.wxml` - 新增搜索框和四大分类数据看板模块

  2. 累计公益推荐官（🤝）：显示总人数，点击展开明细（姓名、注册时间）
  3. 累计已注册单身会员（💑）：显示总人数，点击展开明细（姓名、注册时间）

### 6. 创建数据库 SQL 脚本 - 新建 `visitor_logs` 表
**文件**：`/database/visitor_logs.sql`

主要内容：
- 新建 `visitor_logs` 表，包含字段：
  - `id`: 主键
  - `referrer_id`: 推荐官用户ID（外键关联 `users.id`）
  - `referrer_code`: 推荐码
  - `visitor_openid`: 访客openid
  - `visitor_nickname`: 访客昵称
  - `visitor_avatar`: 访客头像
  - `visit_time`: 到访时间
  - `reg_status`: 注册状态（pending/registered）
  - `created_at`、`updated_at`: 时间戳
- 创建索引：`idx_visitor_referrer`、`idx_visitor_openid`、`idx_visitor_status`、`idx_visitor_time`
- 使用说明：
  1. 当用户通过推荐码/链接进入小程序时，调用 `POST /v1/referral/visitor-log` 记录访客到访行为
  2. 当访客完成注册后，调用 `PUT /v1/referral/visitor-update` 更新 `reg_status` 为 `registered`
  3. 统计未注册访客时，查询 `referrer_id` 下 `reg_status = 'pending'` 的记录
  4. `visitor_openid` 用于去重，同一访客多次访问只记录一次

## 四、后端 API 接口实现

### 1. GET /v1/referral/workbench-stats
**文件**：`routes_referral.js`（行 435-492）

返回当前推荐官四大分类统计数据：
- **联创推荐官**：`user_referrals` 表中 `referrer_id = 当前用户` 且被推荐人 `role = 'partner_matchmaker'` 的用户数
- **公益推荐官**：`user_referrals` 表中 `referrer_id = 当前用户` 且被推荐人 `role = 'public_matchmaker'` 的用户数
- **已注册会员**：`user_referrals` 表中 `referrer_id = 当前用户` 且被推荐人 `role = 'user'` 且存在 `orders.type='single_registration', status='paid'` 的用户数
- **未注册访客**：`visitor_logs` 表中 `referrer_id = 当前用户` 且 `reg_status = 'pending'` 的记录数

### 2. GET /v1/referral/workbench-detail
**文件**：`routes_referral.js`（行 500-656）

支持分页 + 搜索的明细列表，入参：
- `type`：`partner_matchmaker` | `public_matchmaker` | `registered_member` | `visitor`
- `page`：页码（默认1）
- `page_size`：每页条数（默认10，最大50）
- `keyword`：搜索关键词（按昵称模糊匹配）
- 返回：`{ list, total, page, page_size, has_more }`

### 3. POST /v1/referral/visitor-log
**文件**：`routes_referral.js`（行 664-714）

记录访客到访行为（无需登录），通过推荐码解析推荐官ID，同一 `visitor_openid` + `referrer_id` 去重：
- 请求体：`{ referrer_code, visitor_openid, visitor_nickname, visitor_avatar }`

### 4. PUT /v1/referral/visitor-update
**文件**：`routes_referral.js`（行 722-763）

更新访客注册状态（无需登录）：
- 请求体：`{ visitor_openid, reg_status, referrer_code? }`
- `reg_status` 可选值：`registered` | `pending`

### 5. 自动建表机制
`routes_referral.js` 中新增 `ensureVisitorLogsTable(db)` 函数（行 408-428），在每次请求 `workbench-stats`、`workbench-detail(visitor)`、`visitor-log`、`visitor-update` 时自动检查并创建 `visitor_logs` 表和相关索引，无需手动执行 SQL 脚本。

## 五、前端访客日志接入（已完成）

| 接入点 | 文件 | 触发时机 | 调用方法 |
|--------|------|---------|---------|
| 扫码/链接进入小程序 | `app.js` `_handleScene()` | 解析到 `invitationCode` 后保存 pending | 登录成功后由 `_logVisitorIfNeeded()` 自动上报 |
| 登录成功 | `app.js` `login()` | 登录完成后检查 pending 数据 | `referralService.logVisitor()` |
| 首页扫码落地 | `pages/index/index.js` `_handleScanEnter()` | 解析到 `invitationCode` 且有 openid | `referralService.logVisitor()` |
| 支付199元建档成功 | `pages/mine/mine.js` `_doRegistrationPay()` | 支付回调成功 | `referralService.updateVisitorStatus({reg_status:'registered'})` |
| 建档表单提交成功 | `pages/register/register.js` `onSubmit()` | 档案提交成功 | `referralService.updateVisitorStatus({reg_status:'registered'})` |

### 访客日志上报流程

```
用户扫码/点链接进入小程序
  │
  ├─ app.js._handleScene()
  │   ├─ 解析 invitationCode / referrerId
  │   └─ 保存 pending_visitor_code 到 storage
  │
  ├─ 已登录且有 openid → 立即上报 _logVisitorIfNeeded()
  │
  └─ 未登录 → login() 登录成功后
        │
        └─ _logVisitorIfNeeded()
             ├─ 读取 pending_visitor_code
             ├─ 调用 POST /v1/referral/visitor-log
             └─ 上报成功后清除 pending 标记
    
用户注册/建档完成
  │
  ├─ 路径A（我的页面扫码支付199元）
  │   └─ mine.js._doRegistrationPay()
  │        └─ 支付成功 → updateVisitorStatus('registered')
  │
  └─ 路径B（建档表单提交）
      └─ register.js.onSubmit()
           └─ 提交成功 → updateVisitorStatus('registered')
```

## 七、注意事项

1. **权限控制**：已在 `_checkAuth()` 方法中实现权限检查，只有推荐官角色（`public_matchmaker`、`partner_matchmaker`、`city_franchisee`、`professional_recommender`、`community_station`）才能访问该页面
2. **数据隐私**：所有数据仅本人可见，后端接口从 JWT token 中获取 userId，只能查询自己的数据
3. **分页加载**：明细列表默认每页加载10条记录（`PAGE_SIZE = 10`），当 `hasMore = true` 时显示"加载更多"按钮
4. **搜索功能**：支持按姓名搜索，搜索时重置分页参数为第1页
5. **下拉刷新**：使用小程序原生 `scroll-view` 组件实现，刷新时重新加载所有数据
6. **访客去重**：`visitor_logs` 表使用 `visitor_openid` 去重，同一访客多次访问只更新到访时间
7. **自动建表**：后端 API 在首次请求时会自动创建 `visitor_logs` 表，无需手动执行 SQL

## 八、后续优化建议

3. **前端接入**：在首页/扫码落地页接入 `logVisitor()`，在用户注册完成后接入 `updateVisitorStatus()`
