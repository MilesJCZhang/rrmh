## 修复小程序 401 无限循环 + 服务端 /auth/me 404 问题

### 问题根因
1. **服务端 `/auth/me` 路由查询了错误的表**：`authController.me` 查询 `prisma.admin`（管理员表），但小程序用户数据在 `prisma.user` 表。导致 token 验证通过但查不到用户，返回 401。
2. **tsx 运行时 dotenv 路径错误**：`tsx` 的 `__dirname` 行为与 node 不同，`path.resolve(__dirname, "..", ".env")` 指向 `/home/ubuntu/.env`（不存在），导致 `JWT_SECRET` 未加载。
3. **客户端无限循环**：`refreshToken()` 成功后再次 `request()`，如果仍然 401 会再次触发 refresh，形成循环。

### 修复内容
- **服务端 `src/routes/miniAuth.ts`**：将 `/me` 路由从 `authController.me` 改为内联查询 `prisma.user` 表
- **服务端 `src/index.ts`**：dotenv 路径改为尝试多个候选路径（`cwd/.env` 和 `__dirname/../.env`），兼容 tsx 和 PM2
- **客户端 `utils/request.js`**：
  - 添加 `_retry` 标记，同一个请求最多刷新重试 1 次
  - 添加全局刷新锁 `isRefreshing`，防止并发刷新
  - 添加 `silentLogin()` 兜底，refresh 失败时自动静默重新登录

### 技术发现
- `npx tsx` 运行时 `__dirname` 指向项目根目录（不是 `src/` 目录），与 `node` 运行时行为不同
- PM2 的 `cwd` 默认是启动命令时的目录，不是项目目录
