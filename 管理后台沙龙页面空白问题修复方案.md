# 管理后台沙龙活动页面刷新空白 - 修复方案

## 问题描述
管理后台的沙龙活动页面刷新后显示空白，无法看到活动列表。

## 根本原因
前后端状态值不匹配导致API调用失败或数据无法正常显示：

### 1. 状态值不一致
- **前端代码** (`admin-panel/src/pages/Activities/index.tsx`):
  - 默认筛选状态：`pending_review` ❌
  - 状态映射表中使用：`pending_review`, `open`, `full` 等 ❌
  
- **后端代码** (`routes_salon.js`):
  - 创建沙龙时设置状态：`pending_review` ❌
  - 审核通过后状态：`published` ✅
  - 审核拒绝后状态：`rejected` ✅

- **后端API** (`routes_admin_activities.js`):
  - 接受状态参数进行筛选，但值需要与数据库一致

### 2. 数据库中的状态值
根据 `routes_salon.js` 的SQL语句，创建沙龙时状态被设置为 `pending_review`，但后端审核接口期望的是 `pending` 或者直接改为 `published`/`rejected`。

## 待完成的部署步骤

### 方案A：本地构建后上传（推荐）

1. **清理磁盘空间**（当前仅剩14GB）
```bash
# 清理npm缓存
npm cache clean --force

# 删除不必要的文件
cd /Volumes/User/MacBookAir/人人媒好/相亲小程序/miniprogram
rm -rf node_modules/.cache
rm -rf admin-panel/node_modules/.cache
```

2. **重新构建管理后台**
```bash
cd admin-panel
npm run build
```

3. **上传构建后的文件到服务器**
```bash
# 假设服务器IP为 your_server_ip
scp -r build/* ubuntu@your_server_ip:/var/www/admin/
```

4. **重启后端服务**
```bash
# 登录服务器
ssh ubuntu@your_server_ip

# 重启Node.js服务（假设使用PM2）
pm2 restart renrenmeihao-api

# 或者重启Node进程
pkill -f "node.*app.js"
cd /path/to/renrenmeihao-api
nohup node app.js > app.log 2>&1 &
```

### 方案B：在服务器上直接修改（快速修复）

1. **修改服务器上的后端文件**
```bash
# 登录服务器
ssh ubuntu@your_server_ip

# 修改 routes_salon.js，将 pending_review 替换为 pending
sed -i "s/'pending_review'/'pending'/g" /path/to/miniprogram/routes_salon.js

# 重启后端服务
pm2 restart renrenmeihao-api
```

2. **修改服务器上的前端构建文件**
```bash
# 如果构建后的JS文件中有 pending_review 字符串，也需要替换
cd /var/www/admin/static/js
sed -i "s/pending_review/pending/g" main.*.js
```
