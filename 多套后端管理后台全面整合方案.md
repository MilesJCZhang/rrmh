## 第二章：多套代码并行风险分析

### 2.1 运维风险

**R1：多套代码维护负担**

当前同时维护 S1（TS/Prisma）和 S2（JS/better-sqlite3）两套完全不同的代码风格和数据访问层。每次需求变更需在两套代码中分别实现，工作量翻倍。线上用 `dist/` 目录运行编译产物，但 `src/` 和 `dist/` 存在大量 `.bak` 文件（如 `salon.ts.bak`, `salon.ts.bak2`, `salon.ts.bak3`, `salon.ts.broken.*` 共 10+ 个备份），代码版本混乱。

**R2：部署入口不统一**

管理后台既有 `admin-panel/build/`（CRA 构建产物），又有 `app/dist/`（Vite 构建产物），还有 `admin-src/dist/`。Nginx 当前只配置了 `location /admin/` → `/var/www/admin/`，但其他构建产物散落在服务器上，无人维护也不知用途。

**R3：PM2 服务重启频率异常**

PM2 显示 `renrenmeihao-api` 重启次数达 **1131 次**，虽然 uptime 显示当前已运行 9 小时，但高重启次数表明存在频繁崩溃或不稳定。需排查 crash 原因。

**R4：进程端口冲突风险**

本地 S2 监听 3000，线上 S1 监听 3001。如果将来把 S2 部署到服务器，需确保端口不冲突且 Nginx 路由正确分流。

### 2.2 业务风险

**R5：核心商业模式断裂**

S1（线上）缺少三个核心模块：
- **微信支付完整集成**：用户无法付费升级身份，推荐官/合伙人认证的支付流程中断
- **推荐关系绑定（wechat-login）**：新用户注册时无法绑定推荐人，推荐链断裂，佣金无法计算
- **付费后身份自动升级**：即使支付成功，用户身份也不会自动变更，需管理员手动操作

**R6：功能不一致导致用户体验割裂**

S2（本地）包含微信登录推荐绑定、身份升级、手机号绑定、匹配推荐等新功能，但用户完全无法使用（未部署）。小程序 API 配置指向 `https://rrmhdate.cn`（S1），这些新接口返回 404。

**R7：管理后台功能盲区**

A1 管理后台前端有 Archives（档案管理）、Reports（数据报表）等页面，但对应后端可能不完全实现或不返回正确数据。

### 2.3 数据风险

**R8：数据完全割裂**

S1 使用 MySQL `renrenmeihao`（含真实用户数据），S2 使用本地 SQLite `renrenmei.db`（含测试数据）。两个数据库表结构不同、数据不同，无法互通。本地开发时产生的推荐码、支付记录、佣金数据与线上无关。

**R9：关键表缺失导致数据不完整**

MySQL 缺少 `user_referrals` 表，导致推荐关系链无法在数据库层面查询和验证。`referral_relationships` 表虽然存在但可能数据不完整。

**R10：备份机制缺失**

- MySQL `renrenmeihao` 无定时备份脚本
- SQLite `renrenmei.db` 仅为本地文件，无版本控制
- 服务器上存在 `.bak` 文件散落但无系统化管理

## 第三章：整合方案一（统一合并至 MySQL 正式服务）

### 3.1 方案概述

**核心思路：** 以线上 S1（renrenmeihao-api，TypeScript + Prisma + MySQL）为主干，将本地 S2 的独有功能（微信支付、推荐绑定、身份升级、手机号绑定等）全部迁移到 S1，废弃 S2。管理后台只保留 A1，废弃 A3/A4/A5/A6/A8。

**适用场景：** 长期可持续维护的正式方案，一次整合后永久消除多版本问题。

### 3.2 目标架构图

**M1 关键代码迁移示例（auth.ts 补丁 — Prisma 语法）：**

```typescript
// 添加到 src/routes/auth.ts

// 微信登录接口（从本地 routes_auth.js 迁移，改用 Prisma）
router.post('/wechat-login', async (req, res) => {
  try {
    const { code, referrer_id, referral_code } = req.body;
    
    // 1. 调用微信 code2session 获取 openid
    const wxRes = await axios.get('https://api.weixin.qq.com/sns/jscode2session', {
      params: {
        appid: process.env.WECHAT_APPID,
        secret: process.env.WECHAT_SECRET,
        js_code: code,
        grant_type: 'authorization_code',
      },
    });
    
    const { openid, unionid } = wxRes.data;
    if (!openid) throw new Error('微信登录失败：' + (wxRes.data.errmsg || ''));
    
    // 2. 查找或创建用户
    let user = await prisma.user.findFirst({ where: { openid } });
    if (!user) {
      user = await prisma.user.create({
        data: {
          openid,
          unionid: unionid || null,
          nickname: '微信用户',
          role: 'user',
        },
      });
    }
    
    // 3. 绑定推荐关系
    if (referral_code || referrer_id) {
      await bindReferralRelationship(user.id, referral_code, referrer_id);
    }
    
    // 4. 生成 JWT
    const token = jwt.sign(
      { userId: user.id, openid },
      process.env.JWT_SECRET,
      { expiresIn: '30d' }
    );
    
    res.json({ success: true, data: { token, user } });
  } catch (error) {
    res.status(500).json({ success: false, message: error.message });
  }
});

// 推荐关系绑定辅助函数
async function bindReferralRelationship(userId, referralCode, referrerId) {
  // 检查是否已绑定（永久锁定）
  const existing = await prisma.$queryRaw`
    SELECT * FROM user_referrals WHERE user_id = ${userId}
  `;
  if (existing.length > 0) return;
  
  let parentId = referrerId;
  if (referralCode && !parentId) {
    const code = await prisma.referralCode.findFirst({
      where: { code: referralCode, status: 'active' },
    });
    if (code) parentId = code.owner_id;
  }
  
  if (parentId) {
    await prisma.$executeRaw`
      INSERT INTO user_referrals (user_id, parent_id, referral_code)
      VALUES (${userId}, ${parentId}, ${referralCode})
    `;
    // 更新 users 表 parent_id 字段
    await prisma.user.update({
      where: { id: userId },
      data: { parent_id: parentId },
    });
  }
}
```

```typescript
      notify_url: process.env.WECHAT_PAY_NOTIFY_URL,
      trade_type: 'JSAPI',
      openid: req.user?.openid,
    };
    
    // 签名
    params.sign = wxPaySign(params, process.env.WECHAT_PAY_KEY);
    
    // 调用微信统一下单 API
    const xmlBody = objToXml(params);
    const wxRes = await axios.post(

      xmlBody,
      { headers: { 'Content-Type': 'text/xml' } }
    );
    
    const result = xmlToObj(wxRes.data);
    
    if (result.return_code === 'SUCCESS' && result.result_code === 'SUCCESS') {

      const order = await prisma.order.create({
        data: {
          user_id: userId,
          type: orderType,
          amount,
          out_trade_no: params.out_trade_no,
          status: 'pending',
        },
      });

        appId: process.env.WECHAT_APPID,
        timeStamp: String(Math.floor(Date.now() / 1000)),
        nonceStr: params.nonce_str,
        package: `prepay_id=${result.prepay_id}`,
        signType: 'MD5',
      };
      payParams.paySign = wxPaySign(payParams, process.env.WECHAT_PAY_KEY);
      
      res.json({ success: true, data: { payParams, orderId: order.id } });

    delete notifyData.sign;
    if (wxPaySign(notifyData, process.env.WECHAT_PAY_KEY) !== sign) {
      return res.send(failXml('签名验证失败'));
    }
    
    if (notifyData.return_code === 'SUCCESS') {
      const outTradeNo = notifyData.out_trade_no;
      
      // 更新订单状态
      const order = await prisma.order.findFirst({
        where: { out_trade_no: outTradeNo },
      });
      
      if (order) {
        await prisma.order.update({
          where: { id: order.id },
          data: { status: 'paid', paid_at: new Date() },
        });
        
        // 🔑 关键：付费后升级用户身份 + 生成推荐码
        await upgradeUserToPaidTier(order.type, order.user_id);
        
        // 🔑 关键：触发佣金计算
        await calculateCommission(order.id);
      }
      
      res.send(successXml());
    } else {
      res.send(failXml('支付失败'));
    }
  } catch (error) {
    res.send(failXml(error.message));
  }
});
```

**M3 身份升级逻辑（从 S2 迁移）：**

```typescript
// 新增文件: src/services/payment.service.ts

const PAID_TIER_CONFIG = {
  partner_matchmaker: {
    role: 'partner_recommender',
    codePrefix: 'LCRG',
    codeName: '联创推荐官',
  },
  professional_recommender: {
    role: 'professional_recommender', 
    codePrefix: 'ZYRG',
    codeName: '专业推荐官',
  },
  city_franchisee: {
    role: 'city_partner',
    codePrefix: 'CSHH',
    codeName: '城市合伙人',
  },
  community_station: {
    role: 'community_station',
    codePrefix: 'SQZD',
    codeName: '社区服务站',
  },
};

export async function upgradeUserToPaidTier(orderType, userId) {
  const config = PAID_TIER_CONFIG[orderType];
  if (!config) return;
  
  // 1. 生成唯一推荐码
  const code = `${config.codePrefix}${generateRandomCode(4)}`;
  
  // 2. 检查推荐码是否已存在
  const existing = await prisma.referralCode.findFirst({ where: { code } });
  if (existing) {
    // 重新生成
    return upgradeUserToPaidTier(orderType, userId);
  }
  
  // 3. 写入推荐码
  await prisma.referralCode.create({
    data: {
      code,
      type: orderType,
      owner_id: userId,
      status: 'active',
      use_limit: 100,
      use_count: 0,
    },
  });
  
  // 4. 升级用户角色
  await prisma.user.update({
    where: { id: userId },
    data: { role: config.role },
  });
  
  // 5. 更新申请记录状态
  await prisma.$executeRaw`
    UPDATE apply_records SET status = 'approved', reviewed_at = NOW()
    WHERE user_id = ${userId} AND type = ${orderType}
  `;
}
```
