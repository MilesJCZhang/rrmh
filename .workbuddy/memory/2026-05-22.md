## 修复后台管理所有页面数据加载失败问题

### 根本原因
之前执行 `npx prisma db pull` 导致 `schema.prisma` 模型名被覆盖为 snake_case 复数形式，
引发 Prisma Client 生成的关系字段名与实际代码不匹配。

### 修复内容

#### 1. ReferralUsageLog 模型缺少 user 关系字段
- 在 `schema.prisma` 的 `ReferralUsageLog` 模型中添加：
  ```prisma
  user  User?  @relation("ReferralUsageLogUser", fields: [userId], references: [id], onDelete: SetNull)
  ```
- 在 `User` 模型中添加反向关系：
  ```prisma
  referralUsageLogs ReferralUsageLog[] @relation("ReferralUsageLogUser")
  ```

#### 2. 批量修复所有 TS 源码中的关系字段名（PascalCase → camelCase）
| 错误写法 | 正确写法 | 涉及文件 |
|---------|---------|---------|
| `User: true` | `user: true` | admin-users.ts, admin-verification.ts, admin-finance.ts 等 |
| `Wallet: true` | `wallet: true` | admin-users.ts, walletService.ts 等 |
| `Activity: true` | `activity: true` | — |
| `Station: true` | `station: true` | — |
| `Franchisee: true` | `franchisee: true` | — |
| `ReferralUsageLog:` (include) | `usageLogs:` | admin-referral.ts |
| `User_ReferralRelationship_*` | `referee` / `referrer` | admin-referral.ts, referral.ts |
| `frozen_balance` (字段名) | `frozenBalance` | walletService.ts, admin-users.ts |

#### 3. 响应数据映射修复
- `referralCode.User` → `referralCode.referrer`
- `log.User` → `log.user`
- `r.User_ReferralRelationship_*` → `r.referee` / `r.referrer`

#### 4. 重新生成并部署
```bash
npx prisma generate   # 重新生成 Prisma Client
npx tsc              # 编译 TypeScript
pm2 reload renrenmeihao-api --update-env  # 重启服务
```

### 教训
1. **禁止在生产环境执行 `prisma db pull`** — 会覆盖 schema.prisma 中所有模型名和字段名
2. **修改前先创建完整备份** — 本次靠备份才得以回滚
3. **应从源码编译部署，不要直接修改 dist/ 编译后文件**
4. **关系字段名大小写敏感** — Prisma 生成的是 camelCase，不是 PascalCase
