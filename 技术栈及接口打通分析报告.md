小程序端所有API调用都使用 `/v1/...` 前缀，与后端路由挂载路径一致。

**证据**：
1. `utils/config.js` 第18-22行：
   ```javascript
   const API_BASE_URL_MAP = {
     dev: 'http://192.168.31.125:3000',
     prod: 'https://rrmhdate.cn',
   ```

   ```javascript
   let fullUrl = `${API_BASE_URL}${options.url}`;
   ```

   ```javascript

     WECHAT_LOGIN: '/v1/auth/wechat-login',
     REFRESH_TOKEN: '/v1/auth/refresh',

   },
   USER: {
     PROFILE: '/v1/user/profile',
   ```

   - **方案B**：在后端添加 `/api/admin/...` 路径兼容

### 2.3 后端API → 数据库（✅ 已打通）

## 三、问题诊断与修复建议

### 3.1 管理后台崩溃原因（推断）

从之前的对话历史可以看到，用户提到以下问题：
1. "管理后台恢复到6月3日以前的版本"
2. "小程序也崩溃了"
3. "Unknown field 'verification_status'"（Prisma字段名错误）

**可能的原因**：
1. **管理后台构建文件缺失或构建错误**：
   - `admin-panel/build/` 目录可能缺失
   - 或者构建产物不完整

2. **后端路由加载失败**：
   - 从 `server.js` 可以看到，很多路由加载用了 `try-catch`
   - 如果加载失败会跳过，导致API接口缺失

3. **数据库表结构不一致**：
   - `users` 表中可能有字段名不一致的问题
   - 例如：`verification_status` vs `verificationStatus`

4. **API路径不匹配**：
   - 管理后台使用 `/api/admin/...` 前缀
   - 后端没有兼容这个前缀，导致404错误
