# 男女推荐官沙龙活动管理后台完善计划

## 一、当前技术栈总结

### 前端技术栈
- **框架**: React 18.2.0 + TypeScript 4.9.5
- **UI库**: Ant Design 5.12.0 + @ant-design/icons 5.3.0
- **路由**: React Router DOM 6.20.0
- **构建工具**: react-scripts 5.0.1 (Create React App)
- **状态管理**: React Hooks (useState, useEffect, useCallback)
- **HTTP客户端**: axios 1.6.0
- **代理配置**: 开发环境代理到 http://localhost:3000

### 后端技术栈
- **框架**: Express.js (Node.js)
- **数据库**: SQLite 3.x
- **认证**: JWT (requireAuth, requireAdmin中间件)
- **文件上传**: 支持海报图片上传
- **API架构**: RESTful API

### 数据库表结构（沙龙相关）
- **salons**: 沙龙主表（含week_salon_type, max_recommenders, total_cap等字段）
- **salon_groups**: 沙龙分组表
- **salon_group_members**: 沙龙成员表（含companions_json随行人员）
- **weekly_signup**: 每周报名限制表
- **salon_posters**: 沙龙海报表

## 二、男女推荐官沙龙功能现状

### 已实现功能
1. **沙龙类型支持**:
   - `week_salon_type` 字段: 'male' (男推荐官沙龙), 'female' (女推荐官沙龙)
   - 类型标签显示: 蓝色标签(男推荐官), 粉色标签(女推荐官)

2. **容量管理**:
   - `max_recommenders`: 单场推荐官上限9人
   - `max_companions_per_person`: 每人最多随行2人
   - `total_cap`: 单场总人数封顶27人

3. **每周场次管理**:
   - `week_day`: 每周几（1-7）
   - `weekly_signup` 表: 限制每周报名次数

4. **审核流程**:
   - 审核通过/拒绝
   - 拒绝原因必填 (`reject_reason` 字段)
   - 审核通过后生成海报 (`poster_url`)

5. **管理后台展示**:
   - 活动列表显示类型标签
   - 详情弹窗显示一周第几天、容量配置
   - 状态流转: draft → pending_review → published → open → full

### 存在的不足
1. **缺乏专门的男女推荐官沙龙管理界面**: 当前只有一个统一的活动管理页面
2. **推荐官管理功能缺失**: 无法查看推荐官列表、业绩等
3. **随行人员管理不完善**: 只能看到人数，无法查看详情
4. **每周场次管理不直观**: 没有周历视图或场次排期功能
5. **数据统计不足**: 缺乏报名率、参与率、推荐官业绩等数据统计
6. **海报管理功能简单**: 只能查看海报链接，无法预览、下载或批量生成

## 四、技术实施细节

### 前端技术选型
- **状态管理**: 保持使用 React Hooks，复杂场景可考虑 zustand
- **图表库**: 推荐使用 @ant-design/charts 或 echarts-for-react
- **Excel导出**: 使用 xlsx 库
- **日期处理**: 使用 dayjs (Ant Design 已内置)

### 后端API新增需求
1. **推荐官管理API**:
   - `GET /v1/admin/recommenders` - 获取推荐官列表
   - `GET /v1/admin/recommenders/:id` - 获取推荐官详情
   - `GET /v1/admin/recommenders/:id/salons` - 获取推荐官举办的沙龙

2. **场次管理API**:
   - `GET /v1/admin/salons/schedule` - 获取周场次安排
   - `POST /v1/admin/salons/batch-poster` - 批量生成海报

3. **数据统计API**:
   - `GET /v1/admin/dashboard/stats` - 获取仪表盘统计数据
   - `GET /v1/admin/salons/export` - 导出沙龙数据

### 数据库优化建议
1. **添加索引**:
```sql
CREATE INDEX idx_salons_week_salon_type ON salons(week_salon_type);
CREATE INDEX idx_salons_week_day ON salons(week_day);
CREATE INDEX idx_salon_group_members_user_id ON salon_group_members(user_id);
```

2. **考虑分表**:
   - 如果沙龙数据量很大，可以考虑按年份分表
