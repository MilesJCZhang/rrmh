## 修复 /v1/match/recommend 500 错误

### 根因
`authMiddleware` 设置的是 `req.userId` 和 `req.userRole`，但 `avatar.ts` 和 `match.ts` 里读的是 `req.user.userId`（undefined），导致 `Cannot read properties of undefined (reading 'userId')`。

### 修复内容
- `src/routes/avatar.ts`：5处 `req.user.userId` → `req.userId`
- `src/routes/match.ts`：1处 `req.user.userId` → `req.userId`
- `src/routes/commission.ts`：1处 `req.user.role` → `req.userRole`
- 服务器端用 Python 脚本批量替换源码并重新编译/重启 PM2

### 文件清单
| 文件 | 修改内容 |
|---|---|
| `src/routes/avatar.ts` | `req.user.userId` → `req.userId`（×5） |
| `src/routes/match.ts` | `req.user.userId` → `req.userId`（×1） |
| `src/routes/commission.ts` | `req.user.role` → `req.userRole`（×1） |

### 经验教训
- **路由文件中读取用户信息的正确方式**：`req.userId` / `req.userRole`（由 `authMiddleware` 设置）
- **不要使用** `req.user.userId`（undefined）
- 后续新增路由时统一用 `req.userId`，避免再次发生同类问题
