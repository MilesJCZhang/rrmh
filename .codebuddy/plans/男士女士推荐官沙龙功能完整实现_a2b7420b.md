## 产品概述

基于方案文件，为相亲小程序完整实现男士/女士推荐官沙龙功能。该功能仅分两场（男子推荐官沙龙、女子推荐官沙龙），所有已注册推荐官（联创、公益、城市合伙人、社区服务站、专业推荐官）报名权限一致，仅公益推荐官无申办权限，联创推荐官、社区服务站、城市合伙人、专业推荐官均拥有申办场地+主办沙龙+申报主题权限。

## 核心功能

1. **用户性别字段补全**：为 `users` 表新增 `gender` 字段，用于男/女场报名性别限制
2. **每周报名次数限制**：每位推荐官每周仅限报名1次，重复报名自动拦截
3. **人数限额控制**：单场报名推荐官上限9人，每人最多携带2名外部新人，单场总人数封顶27人
4. **随行新人详细资料档案**：主报名推荐官+随行新人需填写姓名（必填）、手机号码（必填），以及性别、年龄、所在行业、身份属性、上班岗位/职务、经营主营项目、个人优势资源简介（选填）
5. **申办权限控制**：联创推荐官、社区服务站、城市合伙人、专业推荐官均可申办沙龙，仅公益推荐官无申办权限（仅可报名参会）
6. **后台审核流程**：用户提交 → 后台人工审核 → 审核通过后自动开启报名端口 + 自动生成活动海报
7. **资料后台归集与导出**：每场沙龙所有报名人员资料自动汇总，后台支持一键导出本场所有人员完整资料表（Excel格式）
8. **海报自动生成**：审核通过场次，系统自动生成标准活动海报（含主题、主办人、时间、地点、小程序报名链接），支持保存、转发、朋友圈引流
9. **自动满员锁场功能**：满27人自动关闭报名端口，显示"本场沙龙已满员"
10. **每周自动重置用户报名次数**

## 技术栈

function getMonday(dateStr) {
  const d = new Date(dateStr);
  const day = d.getDay();
  const diff = d.getDate() - day + (day === 0 ? -6 : 1);
  d.setDate(diff);
  return d.toISOString().split('T')[0];
}

const salon = db.prepare('SELECT * FROM salons WHERE id = ?').get(id);
const weekMonday = getMonday(salon.event_date);

const weeklySignup = db.prepare(
  'SELECT id FROM weekly_signup WHERE user_id = ? AND salon_week_day = ?'
).get(userId, weekMonday);

if (weeklySignup) {
  return res.status(400).json({ code: -1, message: '本周报名次数已用尽' });
}

// 2. 男/女场性别限制
const user = db.prepare('SELECT gender FROM users WHERE id = ?').get(userId);
if (salon.week_salon_type === 'male' && user.gender !== 'male') {
  return res.status(400).json({ code: -1, message: '本场为男士推荐官专属沙龙，仅男士推荐官可报名' });
}
if (salon.week_salon_type === 'female' && user.gender !== 'female') {
  return res.status(400).json({ code: -1, message: '本场为女士推荐官专属沙龙，仅女士推荐官可报名' });
}

// 3. 人数限额校验（27人封顶）
const currentTotal = db.prepare(`
  SELECT SUM(json_array_length(companions_json) + 1) as total 
  FROM salon_group_members 
  WHERE salon_id = ? AND status != 'cancelled'
`).get(id).total || 0;

const newTotal = currentTotal + 1 + (companions ? companions.length : 0);
if (newTotal > salon.total_cap) {
  return res.status(400).json({ code: -1, message: '本场沙龙已满员' });
}

// 4. 记录每周报名
db.prepare(
  'INSERT INTO weekly_signup (user_id, salon_id, salon_week_day) VALUES (?, ?, ?)'
).run(userId, id, weekMonday);

### 七、关键技术细节

#### 7.1 每周报名次数限制实现

**核心思路**：以"本周一日期"作为唯一标识，利用数据库 `UNIQUE(user_id, salon_week_day)` 约束实现幂等性。

**边缘情况处理**：

- 如果用户在周一之前报名下周沙龙，如何计算 week_day？→ 按沙龙 `event_date` 所在周的周一计算
- 如果用户取消报名，是否释放每周名额？→ 是，需在取消时删除 `weekly_signup` 记录

#### 7.2 27人封顶计算逻辑

**公式**：`当前报名主推荐官数 + 所有随行人员数 >= 27` 则拦截

**实现**：

```javascript
// 统计当前总报名人数（含随行）
const countResult = db.prepare(`
  SELECT 
    COUNT(*) as main_count,
    SUM(json_array_length(companions_json)) as companion_count
  FROM salon_group_members 
  WHERE salon_id = ? AND status != 'cancelled'
`).get(id);

const currentTotal = countResult.main_count + (countResult.companion_count || 0);
```
