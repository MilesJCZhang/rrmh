添加了手机号绑定逻辑：
```javascript
// 数据字段
data: {
  phone: '',  // 手机号
  // ...其他字段
}

// 加载用户数据
loadUserData: function() {
  // ...
  const mockData = {
    phone: userInfo.phone || '',  // 加载手机号
    // ...
  };
  this.setData(mockData);
}

// 获取手机号（绑定手机号）
onGetPhone: function(e) {
  if (e.detail.errMsg !== 'getPhoneNumber:ok') {
    wx.showToast({ title: '授权失败', icon: 'none' });
    return;
  }

  const token = wx.getStorageSync('token');
  wx.showLoading({ title: '绑定中...' });

  wx.request({
    url: 'https://你的域名.com/v1/auth/bind-phone',
    method: 'POST',
    header: {
      'Authorization': 'Bearer ' + token
    },
    data: {
      code: e.detail.code
    },
    success: (res) => {
      wx.hideLoading();
      if (res.data.code === 0) {
        wx.showToast({ title: '手机号绑定成功' });
        this.setData({ phone: res.data.data.phone });
      } else {
        wx.showToast({ title: res.data.message || '绑定失败', icon: 'none' });
      }
    },
    fail: () => {
      wx.hideLoading();
      wx.showToast({ title: '网络错误', icon: 'none' });
    }
  });
}
```

## 二、功能流程

### 2.1 扫码注册流程

```
用户扫码
  ↓
小程序 onLaunch/onShow
  ↓
app.js _handleScene(options)
  ↓
获取 query.referrer_id
  ↓
bindReferrer(referrer_id) - 绑定推荐关系
  ↓
wx.login() 获取 code
  ↓
调用后端 /v1/auth/wechat-login
  ↓
后端调用微信 code2Session 获取 openid
  ↓
查询/创建用户（绑定 parent_id）
  ↓
返回 token + 用户信息
  ↓
前端保存 token 和 openid
```

### 2.2 手机号绑定流程

```
用户点击"绑定手机号"按钮
  ↓
微信弹出授权弹窗
  ↓
用户同意授权
  ↓
触发 bindgetphonenumber 事件
  ↓
获取 e.detail.code
  ↓
调用后端 /v1/auth/bind-phone
  ↓
后端获取 access_token
  ↓
调用微信 getuserphonenumber API
  ↓
获取手机号
  ↓
更新用户表的 phone 字段
  ↓
返回成功
  ↓
前端更新 UI 显示手机号
```

## 四、API接口说明

### 4.1 微信登录

**接口**：`POST /v1/auth/wechat-login`

**请求体**：
```json
{
  "code": "微信登录code",
  "referrer_id": "推荐人ID（可选）"
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "token": "JWT Token",
    "openid": "微信openid",
    "userId": 1,
    "phone": "13800138000（如果已绑定）",
    "isNewUser": true
  },
  "message": "登录成功"
}
```

### 4.2 绑定手机号

**接口**：`POST /v1/auth/bind-phone`  
**Header**：`Authorization: Bearer <token>`

**请求体**：
```json
{
  "code": "微信手机号授权code"
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "phone": "13800138000"
  },
  "message": "手机号绑定成功"
}
```

## 六、注意事项

### 6.1 微信接口限制

1. **code2Session**：
   - code 只能使用一次
   - code 有效期5分钟

2. **getPhoneNumber**：
   - 需要用户主动点击按钮触发
   - 需要小程序已认证
   - 每年有调用次数限制（具体看微信文档）

3. **access_token**：
   - 有效期7200秒（2小时）
   - 需要全局缓存，避免重复获取

### 6.2 安全建议

1. **HTTPS**：所有接口必须使用 HTTPS
2. **Token 验证**：所有敏感接口验证 JWT Token
3. **SQL 注入防护**：使用参数化查询（better-sqlite3 已内置）
4. **频率限制**：使用 express-rate-limit 防止暴力破解

## 九、常见问题

### 9.1 微信登录失败

**可能原因**：
1. AppID 或 AppSecret 错误
2. code 已使用或过期
3. 微信服务器网络错误

**解决方法**：
1. 检查环境变量 `WX_APPID` 和 `WX_SECRET`
2. 重新获取 code
3. 检查服务器网络连接

### 9.2 手机号绑定失败

**可能原因**：
1. access_token 无效或过期
2. code 无效
3. 小程序未认证或无权限

**解决方法**：
1. 检查 access_token 是否自动刷新
2. 确认用户已同意授权
3. 在微信公众平台检查权限

### 9.3 推荐关系未绑定

**可能原因**：
1. 扫码参数不正确
2. `referrer_id` 不存在
3. 用户已存在推荐关系（永久锁定）

**解决方法**：
1. 检查小程序码生成时的参数
2. 确认 `referrer_id` 对应的用户存在
3. 查看数据库 `users` 表的 `parent_id` 字段
