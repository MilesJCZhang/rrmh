# 2026-05-31 工作日志

## 推荐码生成规则不一致修复

### 问题
小程序端支付回调生成的推荐码（如 `KWMJ8GG7`、`QMUCFK4G`）使用 `crypto.randomBytes(4).toString('hex').toUpperCase()` 生成 8 位纯随机码，**没有前缀规则**。而后台 `admin-referral.ts` 和 `recommendOfficerController.ts` 使用带前缀规则（如 `LCRG`+4 位随机）。

### 修复内容
1. **`src/routes/payment.ts`**：支付回调生成推荐码逻辑改为带前缀规则，与后台一致
   - 联创推荐官 → `LCRG` + 4 位随机
   - 公益推荐官 → `GYRG` + 4 位随机
   - 专业推荐官 → `ZYRG` + 4 位随机
   - 社区服务站 → `SQZD` + 4 位随机
   - 城市合伙人 → `CSHH` + 4 位随机
2. **数据库修复已有数据**：
   - 用户 60（联创）：`KWMJ8GG7` → `LCRGX1Y2`
   - 用户 88（联创）：`QMUCFK4G` → `LCRGM3N4`
3. **后端已重启**，新规则已生效
