# 沙龙配置管理功能使用说明

## 功能概述

沙龙配置管理功能允许管理员在管理后台动态配置三种沙龙类型（常规沙龙、男推荐官沙龙、女推荐官沙龙）的各项参数，包括主题颜色、功能配置、报名费用、权限设置等。配置会实时同步到小程序端，无需重新发布小程序。

## 使用指南

### 1. 访问沙龙配置管理页面

1. 登录管理后台：`http://rrmhdate.cn/admin/login`
2. 在左侧菜单中找到 **沙龙配置** 菜单项（在"活动管理"下方）
3. 点击进入沙龙配置管理页面

### 2. 配置项说明

沙龙配置包含以下几个部分：

#### 2.1 基本信息

- **类型标识**：沙龙类型的唯一标识（mixed/male_salon/female_salon），不可修改
- **名称**：沙龙名称，如"圈层主题沙龙"
- **描述**：沙龙描述
- **Emoji**：显示的图标
- **状态**：启用/禁用该沙龙类型

#### 2.2 主题配置

- **主题色**：沙龙的主色调，用于按钮、标题等
- **浅色背景**：浅色背景色，用于卡片背景等
- **渐变色**：渐变色，用于Banner背景
- **Banner背景**：Banner背景样式
- **图标**：显示的图标

#### 2.3 功能配置

- **总容量**：每场沙龙的最大参与人数
- **最大推荐官数**：推荐官沙龙的最大推荐官人数
- **每性别最大人数**：常规沙龙的每性别人数（如3男3女）
- **显示评分筛选**：是否显示评分筛选功能
- **允许随行人员**：是否允许携带随行人员
- **最大随行人数**：每人最多携带的随行人数
- **需要资料审核**：是否需要审核用户资料
- **允许现场报名**：是否允许现场报名
- **每周最大场次**：每周最大场次限制

#### 2.4 报名配置

- **需要支付**：报名是否需要支付
- **默认费用**：默认报名费用（元）
- **需要资料**：报名是否需要填写资料
- **允许随行**：是否允许随行人员报名

#### 2.5 权限配置

- **性别限制**：是否限制参与者性别（none/male_only/female_only）
- **最小年龄**：参与者最小年龄
- **最大年龄**：参与者最大年龄
- **最低评分**：参与者最低评分要求

#### 2.6 收益配置

- **推荐官收益(%)**：推荐官收益分成比例
- **平台收益(%)**：平台收益分成比例
- **红娘收益(%)**：红娘收益分成比例

### 3. 编辑沙龙配置

1. 在沙龙配置列表页面，点击 **编辑** 按钮
2. 在弹出的对话框中修改配置项
3. 点击 **确定** 保存配置

### 4. 启用/禁用沙龙类型

在沙龙配置列表页面，使用 **启用/禁用** 开关来控制沙龙类型的状态。

- **启用**：用户可以在小程序端看到并报名参加该类型的沙龙
- **禁用**：用户无法看到该类型的沙龙

### 5. 新增沙龙类型

1. 在沙龙配置列表页面，点击 **新增配置** 按钮
2. 填写基本信息，设置类型标识（如 `new_salon_type`）
3. 配置其他各项参数
4. 点击 **确定** 保存

> **注意**：新增的沙龙类型需要在小程序端添加对应的页面和路由。

### 6. 删除沙龙配置

1. 在沙龙配置列表页面，点击 **删除** 按钮
2. 在确认对话框中点击 **删除**

> **注意**：删除操作不可恢复，请谨慎操作。

## 小程序端集成

### 1. 初始化配置

在小程序的 `app.js` 中调用 `initSalonConfig()` 来初始化配置：

```javascript
// app.js
const { initSalonConfig } = require('./utils/salon-config');

App({
  onLaunch() {
    // 初始化沙龙配置
    initSalonConfig();
  },
});
```

### 2. 使用配置

在页面中使用配置：

```javascript
const { getSalonConfig } = require('../../utils/salon-config');

Page({
  onLoad(options) {
    const salonType = options.type || 'mixed';
    const config = getSalonConfig(salonType);
    
    this.setData({
      themeColor: config.theme.color,
      navTitle: config.name,
    });
  },
});
```

### 3. 强制刷新配置

如果需要强制从后端刷新配置，可以调用 `fetchSalonConfigsFromServer()`：

```javascript
const { fetchSalonConfigsFromServer } = require('../../utils/salon-config');

Page({
  onPullDownRefresh() {
    fetchSalonConfigsFromServer().then(() => {
      wx.stopPullDownRefresh();
      wx.showToast({ title: '配置已更新' });
    });
  },
});
```

## 数据库表结构

`salon_configs` 表结构：

```sql
CREATE TABLE salon_configs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type VARCHAR(50) NOT NULL UNIQUE,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  emoji VARCHAR(10),
  
  -- 主题配置
  theme_color VARCHAR(20),
  theme_light_color VARCHAR(20),
  theme_gradient VARCHAR(200),
  theme_banner_bg VARCHAR(200),
  theme_icon VARCHAR(10),
  
  -- 页面配置
  page_list VARCHAR(200),
  page_detail VARCHAR(200),
  page_create VARCHAR(200),
  
  -- 功能配置（JSON）
  features TEXT,
  
  -- 报名配置（JSON）
  registration TEXT,
  
  -- 权限配置（JSON）
  permissions TEXT,
  
  -- 收益配置（JSON）
  commission TEXT,
  
  -- API配置（JSON）
  api TEXT,
  
  -- 状态
  status TEXT DEFAULT 'active',
  
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 常见问题

### 1. 修改配置后小程序端没有更新？

**解答**：小程序端会在启动时自动从后端获取最新配置。如果需要立即更新，可以：

1. 重启小程序（退出后重新进入）
2. 在下拉刷新时强制刷新配置

### 2. 如何恢复默认配置？

**解答**：可以删除数据库中的配置，然后重新运行 `create_tables.js` 来插入默认配置：

```bash
# 删除配置
sqlite3 renrenmei.db "DELETE FROM salon_configs;"

# 重新插入默认配置
node create_tables.js
```

### 3. 新增沙龙类型后小程序端无法显示？

**解答**：新增沙龙类型需要在小程序端添加对应的页面和路由。请参考现有的沙龙页面创建新页面。
