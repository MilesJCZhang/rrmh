## 第二章：冲突与风险分析

### 2.1 两套服务端并行运行的弊端

| 风险项 | 严重程度 | 具体影响 |
|--------|----------|----------|
| 数据割裂 | 🔴 高 | SQLite用户与MySQL用户完全不通，推荐关系、订单、付费数据割裂 |
| 功能不一致 | 🔴 高 | 本地新增的支付、推荐码功能线上缺失，用户无法实际使用 |
| 迭代维护成本高 | 🟡 中 | 每次更新需要手动同步两份代码（JS版 + TS版） |
| bug复现两次 | 🟡 中 | 同一bug需要在两套代码中分别修复 |
| 数据库语法差异 | 🔴 高 | SQLite与MySQL语法不兼容（如`datetime('now')` vs `NOW()`） |

### 2.2 多套管理后台混乱风险

| 风险项 | 严重程度 | 具体影响 |
|--------|----------|----------|
| 权限不统一 | 🔴 高 | 多套后台可能使用不同的登录认证机制 |
| 操作入口分散 | 🟡 中 | 运营人员需要在多个后台之间切换 |
| 数据不同步 | 🔴 高 | 不同后台可能对接不同的后端服务 |
| 维护成本翻倍 | 🟡 中 | 需要同时维护多套前端代码 |

### 2.3 数据库适配风险

| 风险项 | 严重程度 | 具体影响 |
|--------|----------|----------|
| SQLite不适合生产 | 🔴 高 | 并发性能差，无主从复制，备份困难 |
| 语法差异 | 🔴 高 | 本地代码使用SQLite语法（`datetime('now')`），无法直接迁移到MySQL |
| 数据无法互通 | 🔴 高 | 本地SQLite数据与线上MySQL数据完全隔离 |

### 2.4 部署风险

| 风险项 | 严重程度 | 具体影响 |
|--------|----------|----------|
| 3000端口未启用 | 🟡 中 | 本地新功能无法被小程序访问 |
| 直接切换风险 | 🔴 高 | 如果直接让3000服务上线，会影响现有用户数据 |
| 小程序不切换地址 | 🔴 高 | 前端接口地址未改，用户仍访问3001服务，新功能不生效 |

### 2.5 迭代更新风险

| 风险项 | 严重程度 | 具体影响 |
|--------|----------|----------|
| 手动复制文件易漏改 | 🔴 高 | 本地修改的4个核心模块需要手动同步到线上TS版 |
| 代码版本不一致 | 🟡 中 | 本地JS版与线上TS版逻辑可能逐渐偏离 |
| 测试覆盖不全 | 🟡 中 | 本地测试通过后，线上可能因语法差异出问题 |

#### 方案概述

**步骤1：迁移支付模块（routes_payment.js → payment.ts）**

```typescript
// 在 renrenmeihao-api/src/routes/payment.ts 中新增：

// 1. 导入Prisma客户端
import prisma from '../models/prisma';

// 2. 实现 upgradeUserToPaidTier 函数（适配Prisma语法）
async function upgradeUserToPaidTier(prisma: typeof prisma, orderType: string, userId: number) {
  const config = PAID_TIER_CONFIG[orderType];
  if (!config) return;

  const { role, codeType } = config;

  // 检查用户是否已是该角色（幂等）
  const user = await prisma.user.findUnique({ where: { id: userId } });
  if (!user || user.role === role) return;

  // 生成唯一推荐码
  const CODE_PREFIX_MAP = {
    'creator': 'LCRG', 'professional': 'ZYRG',
    'city_partner': 'CSHH', 'community_station': 'SQZD',
  };
  const prefix = CODE_PREFIX_MAP[codeType] || 'LCRG';
  const chars = '0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ';
  let code, attempts = 0;
  do {
    let suffix = '';
    for (let i = 0; i < 4; i++) {
      suffix += chars.charAt(Math.floor(Math.random() * chars.length));
    }
    code = prefix + suffix;
    const existing = await prisma.$queryRawUnsafe(
      'SELECT id FROM referral_codes WHERE code = ?',
      code
    );
    if (!existing || existing.length === 0) break;
    attempts++;
  } while (attempts < 10);
  if (attempts >= 10) code = prefix + Date.now().toString(36).slice(-4).toUpperCase();

  // 写入 referral_codes 表
  await prisma.$executeRawUnsafe(`
    INSERT INTO referral_codes (code, codeType, referrerId, status, useCount, maxUses, createdAt, updatedAt)
    VALUES (?, ?, ?, 'active', 0, 0, NOW(), NOW())
  `, code, codeType, userId);

  // 升级用户身份
  await prisma.user.update({
    where: { id: userId },
    data: { role, recommendCode: code, referralLevel: 1 }
  });

  // 更新 apply_records
  await prisma.$executeRawUnsafe(`
    UPDATE apply_records SET status = 'approved', updatedAt = NOW()
    WHERE userId = ? AND targetRole = ? AND status = 'pending'
  `, userId, role);
}
```

**步骤2：迁移认证模块（routes_auth.js → auth.ts + authController.ts）**

```typescript
// 在 renrenmeihao-api/src/controllers/authController.ts 中新增：

// 1. 微信登录 wechatLogin
export async function wechatLogin(req, res) {
  const { code, referrer_id, referral_code } = req.body;
  
  // 调用微信 code2Session
  const wxRes = await axios.get('https://api.weixin.qq.com/sns/jscode2session', {
    params: {
      appid: process.env.WX_APPID,
      secret: process.env.WX_SECRET,
      js_code: code,
      grant_type: 'authorization_code',
    }
  });

  const { openid, session_key } = wxRes.data;

  // 查询或创建用户
  let user = await prisma.user.findUnique({ where: { openid } });
  if (!user) {
    // 新用户：写入 parentId + user_referrals 表
    user = await prisma.user.create({
      data: {
        openid,
        parentId: referrer_id || null,
        createdAt: new Date(),
      }
    });

    if (referrer_id) {
      await prisma.$executeRawUnsafe(`
        INSERT INTO user_referrals (userId, referrerId, referralCode, bindTime, isLocked)
        VALUES (?, ?, ?, NOW(), 1)
      `, user.id, referrer_id, referral_code || null);
    }
  } else {
    // 老用户：补填 parentId
    if (!user.parentId && referrer_id) {
      await prisma.user.update({
        where: { id: user.id },
        data: { parentId: referrer_id }
      });

      await prisma.$executeRawUnsafe(`
        INSERT IGNORE INTO user_referrals (userId, referrerId, referralCode, bindTime, isLocked)
        VALUES (?, ?, ?, NOW(), 1)
      `, user.id, referrer_id, referral_code || null);
    }
  }

  // 生成JWT Token
  const token = jwt.sign(
    { uid: user.id, openid: user.openid },
    process.env.JWT_SECRET,
    { expiresIn: '7d' }
  );

  return res.json({
    code: 0,
    data: { token, openid, userId: user.id, role: user.role }
  });
}
```
