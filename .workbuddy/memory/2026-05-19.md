## 修复后台管理登录 404

### 问题描述
手机访问 `rrmhdate.cn`（管理后台登录页），点击登录报错：`HTTP error! status: 404`

### 根本原因
`userApi.ts` 中 `adminLogin()` 使用 `/admin-api/auth/login` 发送请求。
- nginx 将 `/admin-api/` rewrite 为 `/api/admin/`，所以后端收到 `/api/admin/auth/login`
- 但后端只注册了 `/api/auth` 路由（`app.use('/api/auth', authRoutes)`），没有 `/api/admin/auth`
- 所以返回 404

### 修复方案
修改 `adminLogin()`，直接请求 `/api/auth/login`（不走 `/admin-api/` 代理）：
```typescript
// 修改前：
const response = await fetch(`${ADMIN_API_BASE}/auth/login`, ...)
// 修改后：
const response = await fetch(`/api/auth/login`, ...)
```

### 部署
- 本地构建：`npm run build`
- 部署到服务器：`scp` 到 `/var/www/admin/`
- 清理旧 JS 缓存文件

## 修复推荐码管理页面空白

### 问题描述
推荐码管理页面显示空白（暂无数据），但数据库有 10 条推荐码记录。

### 根本原因（三重问题）
1. **请求拦截器读取 token 位置错误**：`request.ts`（axios 拦截器）用 `localStorage.getItem('token')` 读 token，但实际 token 存储在 zustand persist store 中（key: `auth-storage`），导致所有 axios 请求没有 Authorization header → 401
2. **前后端返回格式不匹配**：服务器上部署的是旧版后端代码，`/list` 直接返回数组（`data: [...]`），但前端 `getReferralCodeList` 期望 `{ list, total }`。`res.data.list` 对数组是 `undefined`，所以返回空列表。
3. **浏览器缓存**：nginx 根路径（`location = /`）没有设 `no-cache` 头，浏览器缓存了旧 HTML → 旧 JS 引用 → 加载旧版本代码。

### 修复措施
1. **前端**（`request.ts`）：axios 拦截器改为从 `useAuthStore.getState().token` 读 token，而非 `localStorage.getItem('token')`
2. **前端**（`referralCodeApi.ts`）：`getReferralCodeList` 兼容数组和 `{ list, total }` 两种格式
3. **后端**：重新编译部署 `renrenmeihao-api` 代码（`npm run build` → `scp` → `pm2 restart`）
4. **nginx**：根路径 `location = /` 增加 `Cache-Control: no-cache, no-store, must-revalidate`，防止 HTML 缓存

### 文件修改清单
- `/人人媒好/OKComputer_微信小程序后台设计/app/src/utils/request.ts`
- `/人人媒好/OKComputer_微信小程序后台设计/app/src/lib/referralCodeApi.ts`
- `/etc/nginx/sites-enabled/rrmhdate.cn`（服务器上修改）

- ✅ 通过 nginx `/admin-api/referral-codes/list` 返回相同结果

```sql
(SELECT COUNT(*) FROM single_members smp WHERE smp.userId = users.id) as hasProfile
```
