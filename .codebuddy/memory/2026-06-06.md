## 管理后台用户列表页面修复

### 问题1：TypeError: he.some is not a function（/admin/users 页面空白）
- **原因**：`/users` 和 `/users/:id/status` 路由缺少 `authMiddleware` 鉴权中间件，接口返回 401 导致前端报错
- **修复**：为 `admin.js` 中 `router.get('/users')` 和 `router.put('/users/:id/status')` 添加 `[auth_1.authMiddleware]`

### 问题2：用户列表数据显示不全（ID 1-78 未显示）
- **原因**：`userService.getUserList` 按 `created_at: 'desc'` 排序，最新注册的用户在前，老用户在后
- **修复**：将排序改为 `created_at: 'asc'`（升序），让最早注册的用户显示在第 1 页

### 问题3：前后端分页参数名不匹配
- **原因**：前端发送 `limit` 参数，后端只读取 `pageSize` 参数
- **修复**：`adminController.js` 的 `getUsers` 控制器同时支持 `limit` 和 `pageSize`，优先使用前端传入的 `limit`

### 问题4：Dashboard 页面 TypeScript 编译错误
- **原因**：使用了未定义的 `safeScoreOverview` 变量
- **修复**：改为使用条件渲染后的 `scoreOverview`

### 问题5：user.service.ts API 路径错误
- **原因**：`getUserDetail` 和 `updateUserStatus` 使用 `/api/admin/users` 路径（缺少 `/v1` 前缀）

## 图片上传预览修复 + 保存400错误修复（19:00-19:45）

### 修复列表

#### 1. 图片上传后预览不显示
- **根因**：前端 axios 拦截器返回 `response.data`，但 `ImageUpload.tsx` 用 `res.data?.url || res.url` 提取 URL，实际上正确路径是 `res.data.url`
- **修复**：改为 `res.data?.url || ''`，并直接更新 `fileList` 状态显示预览

#### 2. 保存配置 400 错误（Request failed with status code 400）
- **根因**：后端 `configs` 表的 `updated_at` 字段没有 `@default(now())`，但 `upsert` 的 `create` 部分缺少 `updated_at` 字段
- **修复**：在 `/home/ubuntu/renrenmeihao-api/dist/routes/config.js` 中添加 `updated_at: new Date()`
- 同时修复了同名源文件 `src/routes/config.ts`

#### 3. 推荐码绑定 400 "推荐码无效"
- **根因**：`bindReferrer` 将用户ID当 `recommendCode` 发送，后端只按 `users.recommendCode` 字段查找（如 LCRG001），找不到数字ID
- **修复**：在 `/home/ubuntu/renrenmeihao-api/dist/routes/referral.js` 中，推荐码查找逻辑增加按用户ID（数字）查找
- 先按 `recommendCode` 字段查找，如果没找到且值是纯数字，再按 `users.id` 查找

#### 4. 小程序轮播图 404
- **根因**：旧图片文件已被清理（17797804+ 时间戳的图片），但配置数据库中仍引用旧URL
- **解决**：使用修复后的管理后台上传新图片并保存，即可更新配置

- `admin-panel/src/components/ImageUpload.tsx` - 修复URL提取 + 直接更新fileList
