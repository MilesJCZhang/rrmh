# 2026-06-02 工作记录

## 推荐官工作台功能升级

### 需求概述
用户要求在现有 `matchmaker-workbench` 页面基础上扩展功能，新增专属数据看板，自动统计通过本人专属码/链接引流的全部人员，分类展示累计人数与人员明细。

### 关键决策
1. **页面形式**：在现有 `matchmaker-workbench` 页面基础上扩展（而非新建独立页面）
2. **数据来源**：需要新建数据追踪机制（后端需新建相关表和接口）
3. **访客判定**：扫码/点链接进入但未建档（未支付199元）的用户

### 完成工作

#### 1. 前端改造
- **扩展 `api.js`**：新增4个API端点（`WORKBENCH_STATS`、`WORKBENCH_DETAIL`、`VISITOR_LOG`、`VISITOR_UPDATE`）
- **扩展 `referral.service.js`**：新增4个服务方法（`getWorkbenchStats()`、`getWorkbenchDetail()`、`logVisitor()`、`updateVisitorStatus()`）
- **修改 `matchmaker-workbench.wxml`**：
  - 新增 `<scroll-view>` 容器，支持下拉刷新和上拉加载更多
  - 新增全局搜索框（支持按姓名搜索）
  - 新增四大分类数据看板模块（联创推荐官、公益推荐官、已注册单身会员、未注册访客）
  - 每个分类模块均支持「总数+明细列表」组合形式，点击即可展开查看名单
- **重写 `matchmaker-workbench.js`**：
  - `data` 对象新增工作台相关字段（`workbenchStats`、`searchKeyword`、`expandedType`、`detailList`、`detailPage`、`hasMore`、`isRefreshing`）
  - 新增方法：`onSearchInput()`、`onSearchConfirm()`、`onClearSearch()`、`onToggleDetail()`、`_loadDetailList()`、`onRefresh()`、`onLoadMore()`
  - 修改 `_loadData()` 方法：并发请求收益数据 + 洞察数据 + 工作台统计数据
- **重写 `matchmaker-workbench.wxss`**：
  - 新增样式：搜索框、分类卡片、展开动画、加载更多等

#### 2. 后端准备（需后端开发者配合）
- **创建数据库 SQL 脚本**：`/database/visitor_logs.sql`
  - 新建 `visitor_logs` 表（字段：`id`、`referrer_id`、`referrer_code`、`visitor_openid`、`visitor_nickname`、`visitor_avatar`、`visit_time`、`reg_status`、`created_at`、`updated_at`）
  - 创建索引：`idx_visitor_referrer`、`idx_visitor_openid`、`idx_visitor_status`、`idx_visitor_time`
- **新增API接口**（需后端实现）：

  - `POST /v1/referral/visitor-log`：记录访客到访
  - `PUT /v1/referral/visitor-update`：更新访客注册状态

### 注意事项
1. **权限控制**：已在 `_checkAuth()` 方法中实现权限检查，只有推荐官角色才能访问该页面
2. **数据隐私**：所有数据仅本人可见，后端接口从 token 中获取 userId 进行权限校验
3. **分页加载**：明细列表默认每页加载10条记录，当 `hasMore = true` 时显示"加载更多"按钮
4. **搜索功能**：支持按姓名搜索，搜索时重置分页参数为第1页
5. **下拉刷新**：使用小程序原生的 `scroll-view` 组件实现
6. **访客去重**：`visitor_logs` 表使用 `visitor_openid` 去重，同一访客多次访问只记录一次
