## 需求概述

对人人媒好项目管理后台进行优化升级，确保数据安全的前提下完善管理功能。

## 核心需求

### 1. 数据库备份

- 在优化前自动备份 SQLite 数据库（renrenmei.db）
- 确保小程序平台现有数据不丢失、不删除

### 2. React 管理后台功能完善

- 新增**订单管理**页面：展示线上解锁、沙龙报名、建档等订单数据，支持按类型/状态/日期筛选
- 新增**基金托管**页面：展示用户基金托管记录，支持结算操作（结婚结算/到期退款）

### 3. 小程序管理页面清理

- 删除 `pages/admin/` 目录（主包中的管理页面）
- 删除 `subpackages/user/pages/admin/` 目录（分包中的管理页面）
- 仅删除管理功能代码，不影响小程序核心业务功能

## 功能范围

| 模块 | 操作 | 说明 |
| --- | --- | --- |
| 数据库 | 备份 | 复制 renrenmei.db 为带时间戳的备份文件 |
| 前端路由 | 新增 | App.tsx 增加 /orders 和 /fund-custody 路由 |
| 侧边栏菜单 | 新增 | Layout.tsx 增加订单管理和基金托管菜单项 |

| 托管服务 | 新建 | admin-panel/src/services/fund-custody.service.ts |
| 小程序管理页面 | 删除 | pages/admin/ + subpackages/user/pages/admin/ |
| 构建部署 | 执行 | npm run build 并部署到生产服务器 |


## 非目标（明确不包含）

- 不修改小程序核心业务功能（交友、匹配、推荐等）
- 不修改后端 API（routes_admin_orders.js 已完整，无需改动）
- 不修改数据库表结构
- 不进行大规模重构

## 实施方案

### 1. 数据库备份策略

**方法**：在修改代码前，复制 `renrenmei.db` 为带时间戳的备份文件。

```
# 备份命令（在部署前执行）
cp renrenmei.db renrenmei_backup_$(date +%Y%m%d_%H%M%S).db
```

**备份文件命名规则**：`renrenmei_backup_YYYYMMDD_HHMMSS.db`

**存放位置**：项目根目录（与 renrenmei.db 同级）

## 风险控制

### 数据安全风险

- **风险**：修改过程中可能误删或破坏数据库
- **应对**：修改前必须备份数据库，保留至少3个历史备份

### 部署风险

- **风险**：新版本构建文件上传失败或覆盖错误文件
- **应对**：部署前备份当前生产版本的构建文件，准备回滚方案

### 代码删除风险

- **风险**：误删小程序核心功能代码
- **应对**：只删除 `pages/admin/` 和 `subpackages/user/pages/admin/` 目录，删除后验证小程序编译正常

## 实施顺序

1. **备份阶段**：备份数据库
2. **开发阶段**：创建新页面和服务文件，修改路由和菜单
3. **清理阶段**：删除小程序管理页面代码
4. **构建阶段**：本地构建管理后台
5. **部署阶段**：上传构建文件到生产服务器并重启
6. **验证阶段**：验证新功能可用，确认小程序正常运行

## 设计风格

管理后台采用 **企业级后台管理系统** 设计风格，基于 Ant Design 5.0 设计体系，保持与现有页面一致的视觉风格和交互体验。

## 页面设计

### 1. 订单管理页面（/admin/orders）

**页面布局**（从上到下）：

1. **页面标题区**：使用 PageHeader 组件，标题"订单管理"，副标题"线上解锁+沙龙报名+建档订单"

2. **统计卡片区**：使用 Card + Statistic 组件，展示：

- 线上解锁(优质) 订单数和总金额
- 线上解锁(良好) 订单数和总金额
- 沙龙报名 订单数和总金额
- 会员建档 订单数和总金额

3. **筛选区**：使用 Card + Form/Select/DatePicker 组件，包含：

- 订单类型下拉选择（online_unlock_gold, online_unlock_silver, salon_signup, single_registration）
- 订单状态下拉选择（pending, paid, refunded）
- 日期范围选择器（开始日期、结束日期）
- 查询按钮、重置按钮

4. **表格区**：使用 Table 组件，列包括：

- ID（宽度60）
- 付款人（昵称 + 头像缩略图）
- 订单类型（Tag 标签展示）
- 订单状态（Tag 标签展示）
- 订单金额（红色加粗）
- 创建时间
- 操作（查看详情按钮）

5. **详情弹窗**：使用 Modal + Descriptions 组件，展示订单完整信息

**交互设计**：

- 筛选条件变化后自动刷新表格
- 点击"查看详情"按钮弹出 Modal 展示订单详情
- 表格支持分页（每页20条）

### 2. 基金托管页面（/admin/fund-custody）

**页面布局**（从上到下）：

## 设计规范

### 颜色使用

- **主色调**：Ant Design 默认蓝色（#1677ff）
- **成功色**：绿色（#52c41a），用于"已结算"状态标签
- **警告色**：橙色（#faad14），用于"待结算"状态标签
- **错误色**：红色（#ff4d4f），用于金额显示
- **文本色**：rgba(0,0,0,0.85) 主要文本，rgba(0,0,0,0.45) 次要文本

### 间距规范

- **页面边距**：24px（Content 组件的 padding）
- **卡片间距**：16px（Card 的 marginBottom）
- **表格内边距**：Table 组件默认
- **弹窗内边距**：Modal 组件默认

### 字体规范

- **页面标题**：20px, 600
- **卡片标题**：16px, 500
- **表格内容**：14px, 400
- **辅助文本**：12px, 400

### 组件规范

- **按钮**：使用 Ant Design Button 组件，主操作使用 type="primary"
- **标签**：使用 Ant Design Tag 组件，不同状态使用不同 color 属性
- **表格**：使用 Ant Design Table 组件，开启 pagination，size="middle"
- **弹窗**：使用 Ant Design Modal 组件，宽度400-600px
- **卡片**：使用 Ant Design Card 组件，size="small" 用于统计卡片

## 响应式设计

管理后台主要面向桌面端使用，使用 Ant Design 的响应式栅格系统：

- **≥1200px**：完整展示所有筛选条件和表格列
- **992px-1199px**：筛选条件换行展示
- **<992px**：筛选条件折叠展示（待实现）
