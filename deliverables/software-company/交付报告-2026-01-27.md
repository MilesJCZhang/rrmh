## 四、数据库设计

### 4.1 表结构

#### 4.1.1 users 表（用户）
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  openid TEXT UNIQUE,
  nickname TEXT,
  avatar TEXT,
  phone TEXT,
  gender INTEGER,
  birthday TEXT,
  available_commission DECIMAL(10,2) DEFAULT 0,
  frozen_commission DECIMAL(10,2) DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

#### 4.1.2 stations 表（服务站）
```sql
CREATE TABLE stations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name VARCHAR(100) NOT NULL,
  address TEXT NOT NULL,
  contact_phone VARCHAR(20),
  manager_id INTEGER,
  status TEXT DEFAULT 'active',
  description TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(name)
)
```

**索引**：
- idx_stations_status (status)
- idx_stations_manager (manager_id)
- idx_stations_created (created_at)

#### 4.1.3 partners 表（合伙人）
```sql
CREATE TABLE partners (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL UNIQUE,
  type TEXT NOT NULL CHECK(type IN ('creator', 'public_welfare')),
  status TEXT DEFAULT 'pending',
  level INTEGER DEFAULT 1,
  total_earnings DECIMAL(10,2) DEFAULT 0.00,
  referral_code VARCHAR(20),
  id_card VARCHAR(18),
  id_card_front VARCHAR(200),
  id_card_back VARCHAR(200),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

**索引**：
- idx_partners_user (user_id)
- idx_partners_status (status)
- idx_partners_type (type)
- idx_partners_created (created_at)

#### 4.1.4 withdrawals 表（提现）
```sql
CREATE TABLE withdrawals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  amount DECIMAL(10,2) NOT NULL,
  status TEXT DEFAULT 'pending',
  bank_account VARCHAR(50),
  bank_name VARCHAR(100),
  account_holder VARCHAR(50),
  reject_reason VARCHAR(500),
  processed_at DATETIME,
  processor_id INTEGER,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

**索引**：
- idx_withdrawals_user (user_id)
- idx_withdrawals_status (status)
- idx_withdrawals_created (created_at)

#### 4.1.5 partner_earnings 表（合伙人收益记录）
```sql
CREATE TABLE partner_earnings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  partner_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  order_id INTEGER,
  amount DECIMAL(10,2) NOT NULL,
  type TEXT DEFAULT 'referral',
  description TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(partner_id, order_id)
)
```

**索引**：
- idx_earnings_partner (partner_id)
- idx_earnings_user (user_id)
- idx_earnings_created (created_at)

#### 4.1.6 partner_referrals 表（推荐记录）
```sql
CREATE TABLE partner_referrals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  partner_id INTEGER NOT NULL,
  referred_user_id INTEGER NOT NULL,
  referral_code VARCHAR(20),
  status TEXT DEFAULT 'pending',
  completed_at DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(partner_id, referred_user_id)
)
```

**索引**：
- idx_partner_referrals_partner (partner_id)
- idx_partner_referrals_user (referred_user_id)
- idx_partner_referrals_status (status)

## 五、安全加固

### 5.1 JWT认证中间件

**文件**：`auth-middleware.js`

**功能**：
1. ✅ `generateToken(userId, role, additionalData)` - 生成JWT Token
2. ✅ `verifyToken(token)` - 验证JWT Token
3. ✅ `requireAuth` - 认证中间件（支持开发模式跳过）
4. ✅ `requireAdmin` - 管理员权限中间件
5. ✅ `optionalAuth` - 可选认证中间件

**使用示例**：
```javascript
const { requireAuth, requireAdmin } = require('./auth-middleware');

// 用户认证
router.post('/api/withdrawals', requireAuth, (req, res) => {
  const userId = req.user.userId;
  // ...
});

// 管理员权限
router.get('/api/partners', requireAuth, requireAdmin, (req, res) => {
  // ...
});
```

**开发模式跳过认证**：
```bash
export NODE_ENV=development
export SKIP_AUTH=true
npm start
```

### 5.2 API参数验证

**使用库**：express-validator

**验证示例**：
```javascript
const { body, validationResult } = require('express-validator');

router.post('/apply', [
  body('user_id').isInt({ min: 1 }).withMessage('用户ID必须是正整数'),
  body('type').isIn(['individual', 'company']).withMessage('类型必须是 individual 或 company'),
  body('id_card').optional().isLength({ min: 15, max: 18 }).withMessage('身份证号长度不正确'),
], (req, res) => {
  const errors = validationResult(req);
  if (!errors.isEmpty()) {
    return res.status(400).json({ 
      success: false, 
      errors: errors.array() 
    });
  }
  // 原有逻辑
});
```

**错误响应格式**：
```json
{
  "success": false,
  "message": "参数验证失败",
  "errors": [
    { "field": "user_id", "message": "用户ID必须是正整数" }
  ]
}
```

### 5.3 权限控制

**实现**：基于角色的访问控制（RBAC）

**角色**：
- `user` - 普通用户（默认）
- `admin` - 管理员

**保护的管理端路由**：
- `/api/partners/*` (大部分接口)
- `/api/withdrawals/admin/*` (全部接口)
- `/api/stations/*` (POST, PUT, DELETE)
