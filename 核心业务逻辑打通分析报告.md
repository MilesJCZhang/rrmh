> 分析日期：2026-06-07  
> 分析范围：小程序全量业务模块（用户端 + 管理后台）

### 2.3 未打通的核心模块（❌ 后端完全/部分缺失）

#### 🔴 高优先级 — 核心业务链断裂

**1. 角色申请（APPLY）—— ❌ 缺失率 100%**
- `POST /v1/apply/public-matchmaker` — 申请公益推荐官
- `POST /v1/apply/partner-matchmaker` — 申请联创推荐官（含支付）
- `POST /v1/apply/professional-recommender` — 申请专业推荐官（含支付）
- `POST /v1/apply/city-franchisee` — 申请城市合伙人（含支付）
- `POST /v1/apply/community-station` — 申请社区服务站
- `GET /v1/apply/status` — 申请审核状态查询
- **影响**：用户无法通过小程序申请成为推荐官/合伙人，核心商业模式断裂
- **后端文件**：`routes_apply.js` 不存在

**2. 会员建档（MEMBER）—— ❌ 缺失率 100%**
- `POST /v1/member/register` — 会员建档
- `GET /v1/member/detail` — 会员详情
- `GET /v1/member/list` — 名下会员列表（推荐官用）
- `PUT /v1/member/update` — 更新会员信息
- **影响**：推荐官无法管理名下会员，推荐关系无法产生实际收益
- **后端文件**：`routes_member.js` 不存在

**3. 聊天/AI分身（CHAT + AVATAR + GROUP）—— ❌ 缺失率 100%**
- `routes_chat.js` 不存在（3个端点）
- `routes_group.js` 不存在（3个端点）
- `routes_avatar.js` 存在（2个端点）但**未在 server.js 中挂载**
- **影响**：产品的核心差异化功能（AI聊天画像、群组社交）完全不可用

**4. 推荐官管理（MATCHMAKER）—— ❌ 缺失率 100%**
- 虽然 api.js 注释写着"后端暂未实现"，但 `subpackages/matchmaker/pages/` 下有实际页面
- `GET /v1/matchmaker/status` 等5个端点均无后端实现
- 且 `subpackages/matchmaker/pages/matchmaker/qrcode.js` 中存在硬编码路径 `/v1/matchmaker/qrcode`

#### 🟡 中优先级

**5. 实名认证（VERIFY）—— ❌ 缺失率 100%**
- `POST /v1/verify/submit` — 提交实名认证
- `GET /v1/verify/status` — 认证状态查询
- **后端文件**：`routes_verify.js` 不存在

**6. 支付闭环不完整（PAYMENT）—— ⚠️ 部分实现**
- 已实现：`POST /create`、`POST /notify`
- 未实现：`GET /status/:id`、`POST /refund/:id`、`GET /order/:id`、`GET /orders`
- **影响**：用户无法查询支付状态、无法申请退款

**7. 联谊社交（REUNION + MEET）—— ❌ 缺失率 100%**
- `routes_reunion.js` 不存在（联谊聊天、安排见面）
- `routes_meet.js` 不存在（申请约见）

**8. 文件上传（UPLOAD）—— ❌ 缺失**
- `POST /v1/upload/image`— 上传图片
- `POST /v1/upload/voice`— 上传语音
- 后端无对应路由文件

#### 🟢 低优先级

**9. 统计概览（STATS）—— ❌ 缺失**
- `GET /v1/stats/overview` — 首页统计概览
- **后端文件**：`routes_stats.js` 不存在

## 三、沙龙模块 — 路径不匹配问题（关键）

### 3.1 报名路径不一致

| 定义位置 | 路径 | 问题 |
|---------|------|------|
| `api.js (SALON.JOIN)` | `/v1/salon/:id/join` | 前端实际调用路径 |
| `routes_salon.js` | `POST /:id/signup` | 后端注册路径 |
| 实际挂载后 | `/v1/salon/:id/signup` | 前端调用的 join 会 404 |

**结论**：`join` vs `signup` 路径不匹配，**报名功能实际不可用**。

### 3.2 导出/海报路由未挂载

| 文件 | 路由 | server.js 挂载 |
|------|------|---------------|
| `routes_salon_export.js` | `GET /:id/export` + `POST /:id/poster` | ❌ 未挂载 |

**影响**：管理后台导出报名人员 Excel、生成海报功能不可用。

### 3.3 签到接口缺失

`api.js` 定义了 `API.SALON.CHECKIN: '/v1/salon/:id/checkin'`，但 `routes_salon.js` 中没有 `POST /:id/checkin` 实现。

## 四、管理后台 API 打通分析

### 4.1 API 前缀不统一

管理后台前端13个服务文件的API前缀分布：

| 前缀 | 涉及文件数 | API 数量 | 典型路径 |
|------|-----------|---------|---------|
| `/v1/` | 6 | 19 | `/v1/admin/users`, `/v1/admin/score/rules` |
| `/api/` | 8 | 24 | `/api/admin/partners`, `/api/admin/stations` |

### 4.2 后端兼容性覆盖情况

后端 server.js 为 `/api/` → `/v1/` 做了兼容路由，覆盖情况：

| 管理后台前端路径 | 后端兼容性 | 状态 |
|-----------------|-----------|------|
| `/api/auth/login` | routes_auth 挂载到 `/api/auth`，但 routes_auth 只有 POST `/wechat-login`，没有 POST `/login` | ❌ 不兼容 |
| `/api/auth/logout` | routes_auth 有 POST `/logout` | ✅ 兼容 |
| `/api/admin/orders` | routes_admin_orders 挂载到 `/api/admin` | ✅ 兼容 |
| `/api/admin/partners/*` | routes_partners 挂载到 `/api/admin/partners` | ✅ 兼容 |
| `/api/admin/stations/*` | routes_stations 挂载到 `/api/admin/stations` | ✅ 兼容 |
| `/api/admin/referral-codes/*` | routes_referral-codes 挂载到 `/api/admin/referral-codes` | ✅ 兼容 |
| `/api/admin/salon-configs/*` | routes_admin_salon_config 挂载到 `/api/admin` | ✅ 兼容 |
| `/api/admin/withdrawals/*` | routes_withdrawals 挂载到 `/api/admin/withdrawals` | ✅ 兼容 |
| `/api/admin/fund-custody/*` | routes_admin_orders（含 fund-custody）挂载到 `/api/admin` | ✅ 兼容 |

**主要问题**：管理后台登录接口 `POST /api/auth/login` 不兼容。routes_auth.js 中只有 POST `/wechat-login`（微信静默登录），没有管理员账号密码登录接口。

### 4.3 管理后台重复定义

`fund-custody` 接口在两个服务文件中均有定义：
- `premium.service.ts` → `/v1/admin/fund-custody`
- `fund-custody.service.ts` → `/api/admin/fund-custody`

后端两个路径都已兼容，但存在代码冗余。

## 五、代码质量问题

### 5.1 硬编码 API 路径（12处）

前端代码中存在直接写 `/v1/...` 路径而非通过 `API` 对象引用的文件：

| 文件 | 硬编码路径 | 风险 |
|------|-----------|------|
| `app.js` (3处) | `/v1/user/profile`, `/v1/auth/wechat-login`, `/v1/score/profile` | 后端路径变更时遗漏更新 |
| `utils/referral.js` | `/v1/auth/wechat-login` | 同上 |
| `utils/commissionRules.js` | `/v1/config/public-map` | 同上 |
| `utils/phaseInterfaces.js` (2处) | `/v1/salon/${id}/join`, `/v1/salon/${id}/checkin` | 同上 |
| `utils/contentModeration.js` | `/v1/moderation/text-check` | 路径在 api.js 中不存在 |
| `pages/profile/profile.js` | `/v1/user/profile` | 同上 |
| `pages/register/register.js` | `/v1/user/profile/update` | 同上 |
| `subpackages/matchmaker/pages/matchmaker/qrcode.js` | `/v1/matchmaker/qrcode` | 路径在 api.js 中不存在 |
| `subpackages/matchmaker/pages/matchmaker/earnings.js` | `/v1/referral/my-insight` | 同上 |
| `subpackages/partner/pages/partner-apply/partner-apply.js` | `/v1/user/${id}/avatar-info` | 同上 |

### 5.2 直接调用 wx.request 绕过请求工具

`pages/user/user.js` 和 `subpackages/user/pages/user/user.js` 中直接使用 `wx.request` 而非封装的 `request()` 函数，**缺少 token 注入和 401 自动刷新机制**。

### 5.3 文件存在但未挂载的路由文件

| 路由文件 | 路由数 | 说明 |
|---------|-------|------|
| `routes_avatar.js` | 2 | AI分身信息/活动（mock数据），未挂载到 server.js |
| `routes_salon_export.js` | 2 | 沙龙导出Excel/海报生成，未挂载到 server.js |
| `routes_referral_codes.js`(下划线版) | 7 | 被连字符版 `routes_referral-codes.js` 替代 |

## 七、上线风险评估

### 🟢 可直接上线的部分
- 用户注册登录（认证+资料）
- 推荐关系绑定与推荐码管理
- 评分系统
- 匹配推荐 + 线上解锁
- 高端验资匹配 + 基金托管
- 配置系统
- 管理后台（除登录外的大部分管理功能）
- 沙龙列表/详情/创建/审核/取消/发布/周历

### 🟡 需修复后上线的部分
1. **沙龙报名路径不匹配**：修改 `routes_salon.js` 中 `POST /:id/signup` 为 `POST /:id/join`（或增加兼容路由）
2. **沙龙导出/海报路由挂载**：在 server.js 中追加 `app.use('/v1/salon', require('./routes_salon_export'))`
3. **管理后台登录接口**：在 routes_auth.js 中增加管理员账号密码登录端点
4. **硬编码路径替换**：将12处硬编码路径统一改为 `API` 对象引用

### 🔴 不能上线的部分（核心链路断裂）
1. **角色申请** — 需要创建 `routes_apply.js`（至少6个端点），这是用户成为推荐官的核心入口
2. **会员建档** — 需要创建 `routes_member.js`（至少4个端点），推荐官管理会员的基础
3. **聊天/AI分身** — 需要创建 `routes_chat.js`、`routes_group.js` 并挂载 `routes_avatar.js`

## 九、总结

目前小程序 **约 55% 的核心业务模块已完全打通**，但**三个最关键的业务链路（角色申请、会员建档、聊天/AI分身）处于断裂状态**，其中角色申请和会员建档直接影响商业模式的运转。沙龙模块存在报名路径 404 的严重 bug，需要在发布前修复。管理后台大部分功能可用，但登录接口路径不兼容需要修复。

建议按照第一阶段优先级完成修复后，再进行小程序的正式上线发布。
