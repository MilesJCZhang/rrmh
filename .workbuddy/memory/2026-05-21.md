## 追加：管理后台实名认证接口 500 修复（15:54）

### 现象
管理后台「实名认证」页面无数据，接口 `/api/admin/verifications?status=pending` 返回 500。

### 根因
`admin-verification.ts` 两处 Prisma 关系字段名大小写错误：
- 第 59 行：`include: { user: { select: ... } }` → 应为 `User`（首字母大写）
- 第 137 行：同上
- 第 94-97 行：`item.user?.nickname` 等 → 应为 `item.User?.nickname`
- 第 158-161 行：`singleMember.user?` → 应为 `singleMember.User?`

Prisma schema 中 `SingleMember` 模型的关系字段定义为：
```
User  User  @relation(fields: [userId], references: [id], onDelete: Cascade)
```
字段名是 `User`（首字母大写），代码里用了 `user`（小写），导致：
```
Unknown field `user` for include statement on model `SingleMember`
```

### 修复步骤
1. 服务器上用 Node.js 直接替换 `admin-verification.ts` 中的字段名
2. 编译：`ts.transpileModule` → `dist/routes/admin-verification.js`
3. 重启：`pm2 restart renrenmeihao-api`（pid 3768809）

### 验证
```bash
curl http://localhost:3001/api/admin/verifications?status=pending
# 返回 200，有 1 条 pending 数据
```

### 经验教训
1. **Prisma 关系字段名严格区分大小写**，必须和 schema 定义完全一致
2. **`include` 里的字段名**和**返回数据中的 key** 都是 schema 里定义的名字（首字母大写）
3. 这类 bug 只会报 500，不会报编译错误，需要仔细看 Prisma 错误信息

## 追加：安全修复导致管理后台接口丢失修复（21:56）

### 现象
安全修复后管理后台登录接口返回 400 "用户名或密码错误"，各数据接口也全部 401/404。

### 根因 1：`dotenv` 未加载 → JWT_SECRET undefined → 所有认证接口 401
- `dist/index.js` 没有 `require('dotenv').config()`，`.env` 里的 `JWT_SECRET` 没加载到 `process.env`
- 认证中间件读到 `JWT_SECRET = undefined`，token 验证全部失败

### 根因 2：`stats.js` `/overview` 无认证中间件 → 安全漏洞
- `GET /v1/stats/overview` 之前不需要认证就能访问，公开了用户统计数据

### 根因 3：`admin-finance.js` 弱 fallback → 安全风险
- `const JWT_SECRET = process.env.JWT_SECRET || 'your-secret-key'` — 即使 dotenv 没加载也能用弱密钥验证

### 根因 4：管理后台登录 400 — admin 密码被 shell `$` 替换损坏
- 之前用 SQL 更新密码时，`$2a$10$` 被 shell 替换为 `a0.pzE2vEHSLubbG76th...`，bcrypt 格式被破坏
- `bcrypt.compare` 始终返回 false

### 修复
1. **`dist/index.js`**：首行后加 `require('dotenv').config({ path: '/home/ubuntu/renrenmeihao-api/.env' });`
2. **`dist/routes/stats.js`**：新增 `adminAuth` 中间件，`router.get('/overview', adminAuth, async ...)`
3. **`dist/routes/admin-finance.js`**：`JWT_SECRET` 移除弱 fallback
4. **数据库**：重新 UPDATE admin 密码（双引号包裹 + `\$` 转义），确保 bcrypt 格式完整
5. **PM2 重启**：`pm2 restart renrenmeihao-api`
