# 合规修复说明（2026-05-16）

## 一、修复内容概述

### ✅ 修复1：「一律不退款」条款违反《消费者权益保护法》第25条

**文件**：`pages/agreement/agreement.js`（refund 章节）

**修改前**：
- "本平台服务一经支付成功，即开通对应服务，不支持退款申请"
- 所有付费服务（建档¥199、联创¥399、专业推荐官¥3999、合伙人¥10000）均写"不支持退款"

**修改后**：
- 会员建档：7日内未实际使用核心服务，可申请全额退款
- 联创/专业推荐官：7日内可无理由退款；超过7日扣除实际成本后退还
- 城市合伙人：30日内可申请退款，扣除实际成本后退还
- 沙龙活动：开始前24小时以上可全额退款
- 因平台原因导致无法享受服务的，全额退款

**法律依据**：《消费者权益保护法》第25条——经营者采用网络方式提供服务的，消费者有权在收到服务后7日内无理由要求退款（除非服务已实际提供）。

### ✅ 修复2：隐私弹窗强制同意违反 PIPL 第16条

**文件**：`components/privacy-popup/privacy-popup.js`

**修改前**：
```js
onReject() {
  // ...
  wx.exitMiniProgram();  // 拒绝 = 强制退出，无选择权
}
```

**修改后**：
```js
onReject() {
  // ...
  // 拒绝非必要信息收集，以游客模式进入，限制部分核心功能
  wx.setStorageSync('privacy_guest_mode', true);
  wx.showToast({ title: '已以游客模式进入，部分功能受限', icon: 'none' });
}
```

**配套修改**：`app.js` 中需读取 `privacy_guest_mode` 标记，限制游客模式的功能访问（如禁止智能推荐、禁止聊天等）。

**法律依据**：《个人信息保护法》第16条——个人信息处理者不得以个人不同意处理其个人信息为由，拒绝提供产品或服务（除非该信息是提供服务所必需的）。

#### 3.3 实名认证 API 对接方案（待实施）

**当前状态**：用户手动填写姓名+身份证号，后台人工审核，**未对接公安部接口**。

**风险**：用户可随意填写他人身份证号，平台构成"未尽到身份核验义务"。

**推荐对接方案**：

| 方案 | 服务商 | 功能 | 费用参考 |
|------|--------|------|----------|
| 阿里云：实名认证 | 阿里云 | 姓名+身份证号二要素核验 | ¥0.1/次（批量更便宜）|
| 腾讯云：人脸核身 | 腾讯云 | 姓名+身份证号+活体检测 | ¥1~2/次 |
| 公安部：CTID平台 | 公安部第三研究所 | 最权威，需企业资质 | 需申请对接 |

**推荐实施步骤**（以阿里云为例）：

1. 注册阿里云账号，开通「实名认证」服务
2. 在 `utils/verification.js` 中新增 API 调用：
```js
// utils/verification.js 新增
const ALIYUN_ACCESS_KEY = 'YOUR_ACCESS_KEY';
const ALIYUN_SECRET_KEY = 'YOUR_SECRET_KEY';

export async function verifyIdCard(realName, idNumber) {
  // 调用阿里云实名认证 API
  // 文档：https://help.aliyun.com/document_detail/186190.html
  const res = await wx.request({
    url: 'https://ocr.aliyuncs.com/',
    method: 'POST',
    data: {
      RealName: realName,
      CardNum: idNumber,
    },
    header: {
      'Authorization': 'APPCODE ' + ALIYUN_ACCESS_KEY,
      'Content-Type': 'application/json; charset=UTF-8',
    },
  });
  return res.data;
}
```

3. 在 `pages/verify/verify.js` 的 `onSubmit` 中，先调用核验 API，核验通过后再提交：
```js
async onSubmit() {
  // ... 校验逻辑 ...

  try {
    this.setData({ submitting: true });
    // 第一步：调用公安部实名认证接口核验
    wx.showLoading({ title: '身份核验中...' });
    const verifyResult = await verifyIdCard(realName, idNumber);
    wx.hideLoading();

    if (!verifyResult.IsCorrect) {
      wx.showModal({
        title: '实名认证未通过',
        content: '姓名与身份证号不匹配，请重新确认后提交。',
        showCancel: false,
      });
      return;
    }

    // 第二步：核验通过后，提交到后台
    const result = await submitBasicVerification({ realName, idNumber });
    // ...
  } catch (e) {
    // ...
  }
}
```

4. 后端同步加密存储身份证号（使用 AES-256 或国密 SM4 算法）
